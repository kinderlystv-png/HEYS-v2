# apps/landing — Next.js 14 Landing Rules

## Core Constraints

- **Next.js 14.2** with App Router (`src/app/`)
- **Static export** (`output: 'export'`) — no server-side rendering at runtime
- **Port 3003** for local dev
- **No API routes** — all content from TypeScript config

## File Structure

```
src/
├── app/           # Pages (layout.tsx, page.tsx, legal/)
├── components/
│   ├── sections/  # Landing sections (Pain, HowItWorks, etc.)
│   ├── modals/    # PurchaseModal
│   └── *SSR.tsx   # Pre-rendered components
├── config/        # landing-variants.ts (A/B/C/D content)
├── lib/           # ab-test.ts, useABTest.ts, logger.ts
└── styles/        # globals.css (Tailwind + gradients)
```

## A/B Test Variants

Content is variant-driven. Always pass `variant` and `content` through props:

```typescript
interface Props {
  content: VariantContent; // From landing-variants.ts
  variant: LandingVariant; // 'A' | 'B' | 'C' | 'D'
}
```

- Variants: A (premium), B (direct), C (honest), D (original)
- Config: `src/config/landing-variants.ts`
- Hook: `useABTest(experimentId)` for assignment + persistence

## Styling

- **Tailwind only** — no CSS modules, no inline styles
- Theme colors in `tailwind.config.js`: `heys.primary`, `heys.secondary`, etc.
- Custom classes in `globals.css`: `.gradient-text`, `.hero-gradient`
- Font: Open Sans via `next/font/google`

## SEO & Metadata

Use Next.js Metadata API in layout/page files:

```typescript
export const metadata: Metadata = {
  title: '...',
  description: '...',
  openGraph: { ... },
};
```

## Static Export Gotchas

- `next/image` requires `unoptimized: true` (no server for optimization)
- No `getServerSideProps` / dynamic rendering
- Trailing slashes enabled (`trailingSlash: true`)
- Legal pages under `/legal/` route group with nested redirects

## Dependencies

- `@heys/logger` — workspace logger package
- React 18.2 — functional components + hooks only
