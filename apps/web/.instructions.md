# apps/web — Legacy PWA Rules

## Two Code Worlds

| Layer      | Files                  | Language           | Rule                    |
| ---------- | ---------------------- | ------------------ | ----------------------- |
| **Legacy** | many `heys_*.js` files | Vanilla JS (IIFE)  | Do NOT convert to TS    |
| **Modern** | `src/**`               | TypeScript + React | Strict mode, hooks only |

## Legacy JS Module Pattern

Every `heys_*.js` file follows this IIFE structure:

```javascript
(function (global) {
  'use strict';
  const HEYS = (global.HEYS = global.HEYS || {});
  const U = HEYS.dayUtils || {};
  const M = HEYS.models || {};

  // ... module logic ...

  HEYS.ModuleName = { export1, export2 };
})(window);
```

**Must follow:**

- Always use `HEYS.dayUtils || {}` fallback for dependencies (graceful if
  missing)
- Export to `HEYS.ModuleName`, never to `window` directly
- Use `window.__heysLoadingHeartbeat = Date.now()` for long-loading modules
- Use `HEYS.modulePerf?.startLoad('name')` for performance tracking

## Split Bundles

Legacy code is organized into split bundles (NOT monolithic):

| Directory   | Content                                                     |
| ----------- | ----------------------------------------------------------- |
| `advice/`   | Advice bundles (`_core.js`, `_nutrition.js`, etc.)          |
| `day/`      | Day bundles (`_advice.js`, `_meals.js`, `_meal_quality.js`) |
| `widgets/`  | Widget definitions (`widget_data.js`)                       |
| `insights/` | Predictive insights (15 `pi_*.js` modules)                  |
| `public/`   | Pre-bundled boot scripts (`boot-*.bundle.js`)               |

**Build bundles**: `node scripts/bundle-legacy.mjs` (validates with
`node --check`)

**Critical runtime rule:** after editing any legacy source that affects runtime,
do not stop at source changes only — immediately rebuild the affected runtime
bundle(s) so `localhost` reflects the change right away.

- Prefer selective rebuild:
  `pnpm bundle:legacy:auto --files=<workspace-relative-paths>`
- If bundling config changes (`scripts/bundle-legacy.mjs`,
  `scripts/legacy-bundle-config.mjs`), run full rebuild:
  `pnpm --filter @heys/web run predev && pnpm bundle:legacy`
- Remember: browser runtime uses `apps/web/public/boot-*.bundle.*.js` and
  `postboot-*.bundle.*.js`; rebuilding only `heys_*_bundle_v1.js` is not enough
  for visible UI changes.

## Data Shortcuts (mandatory short forms)

```javascript
dayTot.prot; // NOT dayTot.protein
dayTot.kcal; // NOT dayTot.calories
product.harm; // NOT product.harmScore
```

## localStorage — Scoped by Default

```javascript
U.lsGet('heys_profile', {}); // ✅
U.lsSet('heys_dayv2_2026-03-05', d); // ✅ (v2 prefix!)
localStorage.setItem('key', val); // ⚠️ only for explicit allowlisted unscoped/debug/auth cases
```

Client-scoped data must go through `HEYS.utils.lsSet`/`U.lsSet` or another
contracted wrapper. Direct `localStorage.setItem` is allowed only for
intentional unscoped/debug/auth keys covered by storage guards and allowlists.

## React Bridge (src/)

- `src/App.tsx` bridges legacy HEYS namespace with React components
- `src/main.tsx` is the React entry point
- Components: `CuratorPanel`, `ErrorBoundary`, `LoadingStates`
- Lazy routes in `src/routes/LazyRoutes.tsx`
- React reads global `HEYS.*` — do NOT duplicate state

## Build Pipeline

- Legacy bundles: `scripts/bundle-legacy.mjs` (separate from Vite)
- Selective legacy sync: `pnpm bundle:legacy:auto --files=apps/web/...`
- React + Vite: `vite build` with terser (`drop_console: true` in prod)
- `pnpm dev` starts Vite dev server with HMR on port 3001
- `public/bundle-manifest.json` tracks bundle sizes
