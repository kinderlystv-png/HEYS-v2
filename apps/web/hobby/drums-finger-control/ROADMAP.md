## Как реализовано сейчас

Модуль — `hobby/drums-finger-control/heys_drums_finger_trainer_v1.js`, один IIFE
на 1170 строк, грузится через `heys_day_stats_bundle_loader_v1.js`. Это
отдельный лёгкий hobby-тренажёр, **не** часть большого `fingers/` (тот —
climbing fingerboard на 24 модуля).

**Что есть сейчас:**

- 8 хардкод-блоков (free stroke, rebound, singles, doubles, Moeller, burst,
  buzz, pad flow) → 4 хардкод-сессии (Balanced 25 / Speed 30 / Low Tension 23 /
  Micro 15).
- Fullscreen React-приложение: home (статистика + рекомендация) → run (таймер
  блока, метроном, BPM/зажим/звук/чисто per-блок, pain-чекбокс с алертом) →
  finish (7 метрик сессии + заметка) → saved.
- Лог пишется в `day.trainings[i]` как
  `type:'hobby', hobbySubtype:'drums_finger_control', hobbyLog:{...}`, синк
  через перехватчик LS.
- Резюм прерванной сессии через `heys_drums_finger_active_session`.
- Прогресс — скан LS по `dayv2_`-ключам, эвристика рекомендации (зажим ≥7 →
  мягкая, 2 speed за неделю → баланс).
- Тесты — 4 unit на логику + 1 на таб.

## P0 — баги и архитектурные нарушения

_Чинить первыми. Оценка — **~1 день** с нормальными тестами (client-scope,
resume, edit, UTC/local date, writeDay), а не полдня._

**1. `scanLogs()` нарушает инвариант №9.** Итерирует весь LS по
`key.indexOf('dayv2_')` без фильтра client scope. У куратора с несколькими
клиентами статистика/рекорды/streak смешивают данные всех клиентов. Нужен фильтр
`/^heys_([0-9a-f-]{36})_/` === `currentClientId` (точно как incident 2026-06-02
#13).

**2. `writeDay()` дублирует день в unscoped `heys_dayv2_*`** даже при наличии
scoped-ключа — потенциальная утечка дня одного клиента в base-ключ, который
читают другие. Писать только в scoped, base — только как legacy-fallback без
client id. _Нюанс (подтверждено, строки 239–240): base перезаписывается не
всегда, а только если он пустой или той же даты
(`if (!base || base.date === dateKey)`) — утечка возможна, но не безусловная._

**3. `heys_drums_finger_active_session` не client-scoped** — резюм сессии одного
клиента всплывает у другого. Скоупить ключ + проверить классификацию в storage
registry (синкать или явно исключить).

**4. Streak считается в UTC** (`toISOString().slice(0,10)`), а dateKey дней —
локальный. Для UTC+ вечером streak ложно рвётся. Использовать локальный формат
даты.

**5. `mode:'edit'` не работает.** Pill завершённой тренировки открывает «edit»,
но `makeSessionState` создаёт пустую сессию: прошлые results/metrics не
восстанавливаются. Загрузить `hobbyLog` в state при edit.

**6. Кнопка «Продолжить прерванную сессию» врёт при несовпадении dateKey** —
показывается всегда, но восстанавливает только если дата совпала, иначе молча
стартует новую. Либо резюмить с её dateKey, либо предупредить.

**7. «Мягкий режим» из tension-алерта стирает текущую сессию** (`setSession`
пересоздаёт state) — выполненные блоки и метрики теряются без подтверждения.

**8. `weeklyLimit: 2` у Speed 30 объявлен, но нигде не enforced** — ни
блокировки, ни плашки в UI.

**9. `readActiveSession()` дважды вызывается в render HomeScreen** — чтение LS в
рендере; вынести в state/memo.

**9a. Build/legacy guard (добавлено по ревью).** Модуль живёт в двух бандлах:
`boot-day` грузит `hobby/drums-finger-control/...` (подтверждено в
`boot-day.bundle.*.js`), а таб «Барабаны» — в `postboot-3-ui-lazy.bundle.*.js`
(исходник `heys_training_step_v1.js`). При любой правке P0 прогнать
`pnpm verify:legacy-bundles`, чтобы оба бандла собрались с новым хешем и таб не
отвалился в проде. Это не баг текущего кода, а обязательный шаг приёмки пакета
P0.

## P1 — ядро тренировочного опыта

**10. Метроном на Web Audio lookahead-планировании** вместо `setInterval`
(сейчас джиттер 10–50 мс — для метронома это слышно). Паттерн: scheduler с
`audioContext.currentTime` + очередь нот.

**11. Субдивизии и акценты из паттерна блока:** 8-е/16-е/триоли, акцент каждая
4-я для Moeller-блока, разные звуки акцент/нота/«e&a». Сейчас метроном тикает
только четверти, хотя блоки прямо описывают субдивизии.

**12. Авто-связка таймер+метроном:** count-in 1 такт, метроном стартует со
«Старт» блока, стоп на паузе. Сейчас два независимых тумблера.

**13. Wake Lock API** — экран гаснет посреди блока (руки заняты палками, это
критично для тренажёра).

**14. Голосовые подсказки cue'ов по ходу блока** («не сжимай хват», «второй удар
слышен») — инфраструктура уже есть в `fingers/` (`heys_fingers_voice_v1.js`,
`generate-finger-voice.mjs`), переиспользовать пайплайн. _Приоритет понижен (по
ревью): приятно, но не критичнее точного метронома (10–13) и ramp (15). Делать
после «сердца»._

**15. Tempo ramp внутри блока:** авто-подъём BPM +2–4 каждые N тактов с откатом
по кнопке «грязно» — ключевая механика finger-control тренировки, сейчас BPM
меняется только руками в number-input.

**16. Burst-режим для `burst_8_8`:** метроном сам чередует быстрые/медленные 8
нот (двойной темп на 8 ударов), сейчас блок никак не поддержан механикой.

**17. Большие touch-контролы BPM** (+5/−5/+1/−1, tap-tempo) вместо голого
`<input type=number>`.

**18. Rest-интервалы между блоками** (30–60 сек с бипом) — сейчас блоки встык,
что противоречит методике «полный сброс».

## P2 — прогрессия и методология

**19. Per-block прогрессия по истории:** стартовый BPM блока = последний чистый
BPM из логов (+2 если 2 сессии подряд чисто), а не хардкод 70–110. Поле `metric`
у блоков уже заведено, но не используется — связать блок ↔ метрику сессии
автоматически (BPM из singles-блока → `cleanBpmSingles16` без ручного ввода на
finish).

**20. Встроенный тест one-hand finger taps:** отдельный 20-сек замер на каждую
руку с авто-записью в `oneHandFingerTapBpmRight/Left` (сейчас юзер вводит числа
по памяти).

**21. Недельная периодизация:** лимит speed-сессий (см. №8), авто-deload после
6–7 дней streak или росте avgTension, баланс рук (если left сильно отстаёт —
рекомендовать left-bias сессию). Образец — `fingers/` mesocycle/readiness.

**22. Конструктор сессий:** свой набор блоков/минут/BPM, сохранение пресетов (по
образцу `heys_fingers_constructor_v1.js`). Минимум — настройка минут у штатных
сессий.

**23. Расширение каталога блоков:** paradiddles, accent grids (RLRL с миграцией
акцента), flams, изоляция пальцев (index-only, middle-only), french-grip
one-hand 16ths, slow-motion strokes. Поля уже поддерживают.

**24. Pain-флоу довести до safety-гейта:** сейчас pain — чекбокс + алерт, но
сессию можно продолжать. Добавить: pain → принудительный финиш + рекомендация
паузы 1–2 дня + учёт в `nextSuggestion` (образец `heys_fingers_safety_v1.js` /
pain-gate).

**25. Onboarding-экран для первого запуска:** хват, fulcrum, постановка,
видео/SVG (в `fingers/` есть `onboarding` + `svg_grips` как референс).

## P4 — звук как измерение

_Максимум амбиций. Имеет смысл только после нормального метронома (P1)._ **По
ревью: это не пункт плана разработки, а отдельный research spike.** До оценки
трудозатрат нужно прототипом закрыть риски: permissions (mic/MIDI), шум комнаты,
latency пути «удар → onset», калибровка порога, false positives. Сначала spike →
решение go/no-go → только потом план реализации.

**32. Mic-анализ ровности:** onset detection через Web Audio `AnalyserNode` —
реальная метрика timing evenness и громкостной ровности вместо самооценки 1–5.
Это превращает тренажёр из таймера в измерительный инструмент (главное «по
максимуму»-улучшение).

**33. Авто-замер чистого BPM:** играешь под ramp, mic фиксирует момент развала
тайминга → объективный `cleanBpm`.

**34. MIDI-пад поддержка** (Web MIDI API): для электронных падов — точность до
миллисекунд, велосити каждой руки, авто-все-метрики.
