**Итог 3-го прогона:** в коде живут **две законные линзы** — «план» (без TEF:
норма, долг, осталось) и «физиология» (с TEF: сожжено, факт. дефицит).
Асимметрия TEF, которую 1-й прогон счёл багом, — **by-design и обоснована**
(гипотеза пользователя верна). Реальные баги — там, где линзы протекают или
числа не сходятся внутри одной линзы. Полные вердикты и сводные таблицы — в
THIRD_PASS.md.

Вопрос, на который отвечали: насколько научно и логически согласован алгоритм
энергетического баланса HEYS — от прихода калорий и расхода (TDEE) до нормы и
баланса. Подозрение про несостыковки **подтвердилось**.

## Находки (severity — после 2-го прогона)

| ID   | Находка                                                                                                                       | Sev          |
| ---- | ----------------------------------------------------------------------------------------------------------------------------- | ------------ |
| F-01 | Две формулы BMR одновременно (Mifflin на дне vs Harris-Benedict на профиле, и H-B ошибочно подписан «Mifflin»); Δ 44–101 ккал | **High**     |
| F-02 | Норма делит белок на 4, а приход считает ×3 → недобор 104–172 ккал. (Сам ×3 — корректная net-energy модель, НЕ баг)           | **Med-High** |
| F-03 | carb/fat TEF вне таргета (скрытый мини-дефицит на поддержании); ярлык «белок 25%» при факте 0                                 | **Medium**   |
| F-04 | Активность по gross MET поверх 24-часового BMR → двойной счёт покоя; Δ 30–60/трен                                             | Medium       |
| F-05 | Три формулы шагов, шаг 1.19 м завышен (+50–70%), рост зашит 170                                                               | Medium       |
| F-06 | Поле пола: `gender` в профиле vs `sex` в коде → молчаливый дефолт «мужчина» (latent)                                          | Medium       |
| F-07 | Исторический/календарный TDEE без TEF и NDTE, живой — с ними                                                                  | Medium       |
| F-10 | EPOC (advisory) + NDTE (применён) — дубль послетренировочного дожига; 4-я MET-константа                                       | Medium       |
| F-08 | NDTE-надбавка к расходу — низкая доказательность                                                                              | Low          |
| F-09 | Смешение additive- и multiplier-моделей TDEE                                                                                  | Low          |
| F-11 | TEF% в UI противоречит совету «добавь белка»                                                                                  | Low          |

Корень один: **нет единого вычислительного ядра** — BMR в 2 формулах, MET→ккал в
4 константах, TDEE в 3 моделях; две осознанные научные идеи (net-energy белка;
optimum без TEF) применены только к одной стороне уравнения.

Главное по вопросу пользователя: **белок 3 ккал/г — научно обоснован (TEF белка
вшит) и не является ошибкой.** Чинить нужно консистентность, а не сам белок.

Диагностика, правок в код не вносилось. Численная сверка — на реальных модулях
(`verify_caloric.cjs`), результаты в [SECOND_PASS.md](./SECOND_PASS.md).
