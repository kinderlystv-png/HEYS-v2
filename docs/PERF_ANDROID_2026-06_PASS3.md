## 3. Дополнения к плану PASS2

### В Блок A (нулевой риск, визуал идентичен)

- **A7. Ступенчатая загрузка postboot-чанков.** В фасадах поднять `timeout` RIC
  с 2000 до ~10000 и грузить чанки **последовательно** (следующий — после parse
  предыдущего + реальный idle), а не все разом. Поведение «клик по вкладке =
  мгновенный force-load» уже существует (`HEYS.__loadPostboot1Game` для
  React.lazy) — UX перехода не меняется, исчезает только фоновый парс-шторм в
  первые секунды. Файлы: `heys_game_facade_v1.js`,
  `heys_postboot3_facade_v1.js`, insights-фасад. Откат: вернуть константу.
- **A8. key={index} → key={id/type}** в `heys_day_meals_bundle_v1.js:1678,1687`.
- **A9. Build-hash в log-trace.** При буте логировать хеш бандла (из
  `bundle-manifest.json`, доступен сборке) — однострочник; даёт ответ «какой код
  реально исполняется у жалующегося клиента». Риск нулевой.

### В Блок B (малый риск, под флагом)

- **B5. Поиск продуктов: бюджет скоринга.** Ранний выход по длине запроса (<2
  символов — только prefix), cap кандидатов для fuzzy (например top-300 по
  prefix), мемо результата по строке запроса. Видимый порядок выдачи для типовых
  запросов не меняется — проверить снапшот-тестом выдачи на фиксированном
  каталоге. Web Worker — НЕ сейчас (блок C).
- **B6. Ввод граммов: подтвердить замером (блок 0) долю setDay-каскада** в
  спайке на символ. Если значима — батчить коммит граммов (state локально в
  инпуте, setDay по blur/250 мс idle). Это изменение поведения данных (момент
  записи) → только под флагом и после INP-базлайна.
- **B7. Проверка стабильности identity** 9 deps-колбэков `app_hooks:1762`
  (console.count в dev на re-register). Если churn есть — стабилизировать
  useCallback'и; если нет — закрыть без правки.

### В Блок C (пострелиз)

- Decompress-кэш по строке для путей мимо Store.get (вместе с B1 single-parse —
  одна структура кэша на interceptor).
- Web Worker для fuzzy-поиска и compress/decompress больших каталогов.

Блок 0 (INP-базлайн) из PASS2 становится ещё важнее: P3-2 и B6 без замера не
решаются — доли interceptor vs React-каскада в спайке на символ неизвестны.
