## 2. Цель обработки персональных данных

```
Предоставление пользователям сервиса HEYS услуг по сопровождению питания
и контролю здоровья, включая ведение электронного дневника питания,
автоматический расчёт калорий и БЖУ, аналитику динамики веса и активности,
персональные рекомендации, взаимодействие с куратором, обеспечение работы
личного кабинета и связь с пользователями.
```

## 3. Правовое основание обработки

```
- ст. 6 ч. 1 п. 1, 5 Федерального закона от 27.07.2006 № 152-ФЗ
  (согласие субъекта; исполнение договора, стороной которого является субъект ПДн)
- ст. 9 Федерального закона от 27.07.2006 № 152-ФЗ
  (согласие в письменной форме / электронной с использованием ПЭП)
- ст. 10 Федерального закона от 27.07.2006 № 152-ФЗ
  (специальная категория ПДн — данные о состоянии здоровья — обрабатывается
  только при наличии письменного согласия субъекта)
- Гражданский кодекс РФ (заключение и исполнение договора оказания услуг)
- Налоговый кодекс РФ (учётные обязанности оператора)
```

## 4. Категории субъектов персональных данных

```
- Клиенты сервиса — физические лица, использующие приложение для ведения
  дневника питания и получения услуг сопровождения
- Кураторы — физические лица, оказывающие услуги сопровождения клиентам
  в рамках платных тарифов
- Посетители сайта heyslab.ru, оставившие заявку через форму на лендинге
  (потенциальные клиенты)
```

## 5. Перечень обрабатываемых персональных данных

**Общие категории:**

```
- Фамилия, имя, отчество
- Номер мобильного телефона
- Адрес электронной почты
- Дата рождения
- Пол
- Рост, вес и динамика веса
- Изображения (аватар профиля — опционально)
```

**Специальная категория ПДн (данные о состоянии здоровья):**

```
- Сведения о приёмах пищи, составе и количестве потребляемых продуктов
- Сведения о физической активности
- Сведения о сне и режиме дня
- Сведения о менструальном цикле (для пользователей женского пола)
- Сведения о самочувствии и общем состоянии
- Расчётные показатели: калорийность, БЖУ, ИМТ, водный баланс
```

**Технические данные:**

```
- IP-адрес устройства пользователя
- Идентификатор браузера (User-Agent)
- Сведения об устройстве, операционной системе
- Дата и время использования сервиса
- Действия пользователя в приложении (логи)
```

## 6. Действия с персональными данными

```
Сбор, запись, систематизация, накопление, хранение, уточнение (обновление,
изменение), извлечение, использование, передача (предоставление, доступ),
обезличивание, блокирование, удаление, уничтожение.
```

## 7. Способы обработки

```
- Смешанная обработка с использованием средств автоматизации
- Обработка осуществляется в информационной системе персональных данных,
  размещённой на территории Российской Федерации (Yandex Cloud,
  ЦОД Москва/Владимир)
- Передача персональных данных по сети Интернет осуществляется
  с использованием защищённых протоколов (HTTPS/TLS 1.3)
- Трансграничная передача персональных данных не осуществляется
```

## 8. Меры по обеспечению безопасности ПДн

```
Применяются правовые, организационные и технические меры в соответствии
со ст. 18.1 и ст. 19 152-ФЗ:

Правовые:
- Назначен сотрудник, ответственный за организацию обработки ПДн (приказ № 1)
- Утверждены локальные документы по обработке ПДн (Положение об обработке)
- Получаются письменные согласия субъектов на обработку (включая ПЭП через SMS
  для специальной категории — данных о здоровье)

Организационные:
- Контроль доступа: только сам ИП и привлечённые на основании договора
  кураторы имеют доступ к данным клиентов в объёме, необходимом для оказания
  услуг
- Логирование действий с ПДн в системе

Технические:
- Шифрование передачи данных (HTTPS, HSTS)
- Шифрование хранилища (Yandex Cloud Postgres encryption-at-rest)
- Хеширование PIN-кодов с использованием соли
- Ограничение количества попыток ввода PIN с блокировкой
- Резервное копирование с шифрованием
- Веб-уровень: Content Security Policy, X-Frame-Options, X-Content-Type-Options
- Мониторинг событий безопасности
```

## 10. Срок или условия прекращения обработки

```
Обработка ПДн прекращается:
- по достижении целей обработки;
- при отзыве субъектом ПДн согласия на обработку (для случаев, когда
  обработка осуществляется на основании согласия);
- при удалении субъектом ПДн своего аккаунта в сервисе;
- по решению уполномоченного органа;
- при ликвидации деятельности оператора.

После прекращения обработки ПДн уничтожаются в срок, не превышающий 30 дней,
если иное не установлено законодательством РФ.
```
