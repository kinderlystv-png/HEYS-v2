## 2. Цель обработки персональных данных

```
Предоставление законным представителям несовершеннолетних услуг по
организации и сопровождению детских мероприятий (праздников, дней рождения),
включая планирование мероприятий, координацию услуг подрядчиков, ведение
расписания и меню, формирование пригласительных, обеспечение работы личного
кабинета клиента, связь с законным представителем по вопросам организации
мероприятия.
```

## 3. Правовое основание обработки

```
- ст. 6 ч. 1 п. 1, 5 Федерального закона от 27.07.2006 № 152-ФЗ
  (согласие субъекта; исполнение договора)
- ст. 9 Федерального закона от 27.07.2006 № 152-ФЗ
  (согласие в письменной/электронной форме)
- Гражданский кодекс РФ, ст. 435, 437, 438 (публичная оферта; акцепт)
- Гражданский кодекс РФ (заключение и исполнение договора)

Обработка данных несовершеннолетних осуществляется исключительно на основании
согласия их законного представителя (родителя/опекуна) в соответствии
с законодательством РФ о защите прав ребёнка.
```

## 4. Категории субъектов персональных данных

```
- Законные представители несовершеннолетних (родители, опекуны), являющиеся
  клиентами платформы Kinderly
- Несовершеннолетние участники организуемых мероприятий — обработка
  осуществляется на основании согласия законного представителя
- Подрядчики (аниматоры, фотографы и иные исполнители), привлекаемые для
  оказания услуг — при наличии договора-поручения
- Посетители сайта, оставившие заявку через форму обращения
```

## 5. Перечень обрабатываемых персональных данных

**Данные законных представителей:**

```
- Фамилия, имя, отчество
- Номер мобильного телефона
- Адрес электронной почты
- Сведения об оплате услуг (без полных реквизитов банковских карт —
  расчёты производятся внешними способами; платёжная инфраструктура
  Kinderly хранит только статусы транзакций)
```

**Данные несовершеннолетних** (обрабатываются с согласия законного
представителя):

```
- Имя ребёнка
- Возраст ребёнка
- Предпочтения по тематике мероприятия, меню
```

**Данные мероприятий:**

```
- Дата, время, место проведения мероприятия
- Количество гостей
- Тематика, оформление, меню, требования по фото/видео
```

**Технические данные:**

```
- IP-адрес устройства
- User-Agent
- Cookies сессии (session_token)
- Сведения об устройстве и браузере (device fingerprint)
- Логи входа в личный кабинет
```

## 6. Действия с персональными данными

```
Сбор, запись, систематизация, накопление, хранение, уточнение (обновление,
изменение), извлечение, использование, передача (предоставление, доступ
подрядчикам по договору-поручению), обезличивание, блокирование, удаление,
уничтожение.
```

## 7. Способы обработки

```
- Смешанная обработка с использованием средств автоматизации
- Обработка осуществляется в информационной системе персональных данных,
  размещённой на территории Российской Федерации (Yandex Cloud,
  ЦОД Москва/Владимир)
- Передача по сети Интернет осуществляется с использованием защищённых
  протоколов (HTTPS/TLS 1.3)
- Трансграничная передача персональных данных не осуществляется
```

## 8. Меры по обеспечению безопасности ПДн

```
Применяются правовые, организационные и технические меры в соответствии
со ст. 18.1 и ст. 19 152-ФЗ:

Правовые:
- Назначен ответственный за организацию обработки ПДн (приказ № 1)
- Утверждено Положение об обработке персональных данных
- Получается согласие законного представителя несовершеннолетнего на
  обработку данных ребёнка
- При привлечении подрядчиков (аниматоры, фотографы) заключаются
  договоры-поручения об обработке ПДн в порядке ч. 3 ст. 6 152-ФЗ

Организационные:
- Контроль доступа: персонификация учётных записей администраторов
- Принцип минимизации данных (подрядчикам передаётся только необходимый
  минимум сведений)
- Логирование действий и согласий

Технические:
- HTTPS, HSTS, SameSite=Strict cookies
- CSRF-токены и защита от автоматизированных запросов (Cloudflare Turnstile)
- Rate limiting на формы входа
- Шифрование БД (Yandex Cloud Postgres encryption-at-rest)
- Защита приложения на уровне Row-Level Security в базе данных
```

## 10. Срок или условия прекращения обработки

```
Обработка ПДн прекращается:
- по достижении целей обработки (завершение организации мероприятия);
- при отзыве субъектом ПДн (либо законным представителем) согласия;
- при удалении аккаунта клиента;
- по истечении срока хранения, установленного локальными актами;
- по решению уполномоченного органа.

После прекращения обработки ПДн уничтожаются в срок, не превышающий 30 дней.
```

## 12. Сведения о трансграничной передаче

```
Трансграничная передача персональных данных не осуществляется.

Примечание: до завершения миграции с Supabase (см. план P0-0) часть данных
исторически могла располагаться на инфраструктуре за пределами РФ. После
завершения миграции все ПДн хранятся исключительно на территории Российской
Федерации.
```

> ⚠️ Если на момент подачи уведомления миграция с Supabase **не завершена** —
> либо ускорить её перед подачей, либо в этом разделе указать «трансграничная
> передача осуществляется» с описанием. Лучший вариант: завершить P0-0 → подать
> уведомление → указать «передача не осуществляется».
