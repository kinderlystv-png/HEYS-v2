# HEYS · Data-change gate

Статус: R0 ready template. Использовать перед любой фичей, которая трогает
профиль, дневник, health-сигналы, лиды, платежи, аналитику, кураторский доступ
или интеграции.

## Pre-release строка

Перед релизом фичи в changelog/release review должна быть строка:

> ПДн data-change gate пройден: новых категорий ПДн / целей / получателей /
> сроков / трансграничной передачи нет; либо ссылки на обновлённые документы и
> blocking-задачи указаны.

## 5 вопросов

| Вопрос                                                    | Да/нет | Что сделать при «да»                                                    |
| --------------------------------------------------------- | ------ | ----------------------------------------------------------------------- |
| Добавляется новая категория ПДн или health-сущность?      |        | Обновить `heys-data-register.md`, privacy/health consent, проверить РКН |
| Меняется цель обработки?                                  |        | Обновить документы и РКН-драфт/изменение                                |
| Появляется новый получатель/обработчик?                   |        | Обновить access matrix/subprocessors, договор/поручение, документы      |
| Появляется зарубежный сервис или трансграничная передача? |        | Отдельная проверка ст. 12 152-ФЗ до включения                           |
| Меняются сроки хранения, доступы, логи или удаление?      |        | Обновить регламент, audit/delete flow, monthly audit checklist          |
