## 💰 Caloric Debt (Калорийный долг)

**Файл**: `heys_day_caloric_balance_v1.js` (ранее `heys_day_v12.js`) |
**useMemo**: `caloricDebt`, `displayOptimum`, `displayRemainingKcal`

### Концепция

Система автоматически отслеживает **недобор калорий за последние 3 дня** и
корректирует сегодняшнюю норму, чтобы помочь восполнить дефицит. Это
предотвращает:

- Хронический недоед → срывы
- Замедление метаболизма
- Потерю мышечной массы при дефиците

### Параметры расчёта v2.0 — Гибкое восстановление

> 🔬 **Научное обоснование:**
>
> - Leibel et al., 1995 (PMID: 7632212): Метаболизм адаптируется на ~15% при
>   дефиците → **не нужно компенсировать 100% долга**
> - Hall et al., 2011 (PMID: 21872751): Постепенные изменения эффективнее резких
>   → **гибкие 1-3 дня вместо фиксированных 4**
> - Rosenbaum & Leibel, 2010 (PMID: 20107198): При хроническом дефиците резкое
>   увеличение калорий может откладываться в жир

| Константа         | Значение | Описание                                        |
| ----------------- | -------- | ----------------------------------------------- |
| `DEBT_WINDOW`     | 3        | Окно анализа (дней назад)                       |
| `MAX_DEBT`        | 1500     | Максимум учитываемого долга (ккал)              |
| `RECOVERY_TARGET` | 0.75     | **Компенсируем только 75% долга** (Leibel 1995) |
| `MAX_BOOST_PCT`   | 0.20     | Максимум +20% к норме в день                    |
| `TRAINING_MULT`   | 1.3      | Недобор в тренировочный день ×1.3 (критичнее)   |

### Гибкие дни восстановления

Количество дней зависит от размера долга:

| Долг (ккал) | Дней на восстановление | Обоснование                  |
| ----------- | ---------------------- | ---------------------------- |
| < 300       | 1 день                 | Малый долг — закрыть быстро  |
| 300-700     | 2 дня                  | Средний — плавно без стресса |
| > 700       | 3 дня                  | Большой — не перегружать ЖКТ |

**Формула**: `dailyBoost = (debt × 0.75) ÷ recoveryDays`

**Пример**: Долг 600 ккал → effectiveDebt = 450 ккал → за 2 дня → +225 ккал/день

## 🔴 Caloric Excess (Мягкая коррекция перебора)

**Файл**: `heys_day_v12.js` | **Версия**: 1.0.0 (2025-12-17)

### 🧠 Философия — НЕ наказание!

> **Ключевой принцип**: Строгие ограничения провоцируют срывы (Herman & Polivy
> 1984). HEYS использует МЯГКУЮ коррекцию + акцент на АКТИВНОСТИ.

| Подход           | Эффект         | В HEYS                       |
| ---------------- | -------------- | ---------------------------- |
| ❌ "Наказание"   | Срывы, стресс  | НЕ используется              |
| ✅ Активность    | Здоровая норма | **70% компенсации**          |
| ✅ Мягкий акцент | Напоминание    | **5-10% снижение нормы max** |

### Константы (EXCESS_CFG)

| Константа                | Значение | Описание                             |
| ------------------------ | -------- | ------------------------------------ |
| `SOFT_REDUCTION_PCT`     | 0.05     | **5%** — мягкая коррекция            |
| `MODERATE_REDUCTION_PCT` | 0.08     | **8%** — умеренная                   |
| `MAX_REDUCTION_PCT`      | 0.10     | **10%** — максимум (НИКОГДА больше!) |
| `ACTIVITY_PRIORITY`      | 0.70     | **70%** через активность (главное!)  |
| `SOFT_THRESHOLD`         | 200      | <200 ккал — игнорируем               |
| `MODERATE_THRESHOLD`     | 400      | 200-400 — мягкая коррекция           |
| `MAX_RECOVERY_DAYS`      | 2        | Максимум 2 дня (не растягиваем)      |

## 🔄 Refeed Day (Загрузочный день)

**Файл**: `heys_refeed_v1.js` | **Версия**: 1.3.0

### Концепция

Refeed Day — **осознанный** день повышенного калоража для восстановления
метаболизма. Это НЕ срыв, а часть стратегии! Отмечается пользователем в утреннем
чек-ине с выбором причины.

### Константы

| Константа            | Значение | Описание                         |
| -------------------- | -------- | -------------------------------- |
| `REFEED_BOOST_PCT`   | 0.35     | +35% к норме калорий             |
| `REFEED_THRESHOLD`   | 1000     | Порог долга для рекомендации     |
| `REFEED_CONSECUTIVE` | 5        | Дней в дефиците для рекомендации |
| `REFEED_OK_RATIO`    | 1.35     | Допустимый перебор (135%)        |

### Причины refeed дня

| ID         | Иконка | Название                      | Описание                           |
| ---------- | ------ | ----------------------------- | ---------------------------------- |
| `deficit`  | 💰     | Восстановление после дефицита | Накопился долг калорий             |
| `training` | 💪     | После интенсивной тренировки  | Нужно восстановить гликоген        |
| `holiday`  | 🎉     | Праздник / особый день        | Запланированное превышение         |
| `rest`     | 🧘     | Ментальный отдых от диеты     | Снятие психологического напряжения |

### Refeed Zones (зоны выполнения)

| Зона    | Ratio (kcal/optimum) | Описание                  | Цвет       |
| ------- | -------------------- | ------------------------- | ---------- |
| `ok`    | 0.90 – 1.35          | Целевая зона              | 🟢 Зелёный |
| `under` | < 0.90               | Недобор (не refeed!)      | 🟡 Жёлтый  |
| `over`  | 1.35 – 1.50          | Умеренный перебор         | 🟠 Оранж.  |
| `binge` | > 1.50               | Срыв (не refeed, а binge) | 🔴 Красный |

### Поля в DayRecord

| Поле           | Тип     | Описание                                   |
| -------------- | ------- | ------------------------------------------ |
| `isRefeedDay`  | boolean | Отмечен ли день как загрузочный            |
| `refeedReason` | string  | ID причины (deficit/training/holiday/rest) |

### API

```javascript

HEYS.Refeed.shouldRecommendRefeed(caloricDebt); // Рекомендовать ли refeed
```

## 🍽️ Meal Quality Score (2025-12-10)

**Файл**: `heys_day_v12.js` | **Функция**: `getMealQualityScore()` | **Шкала**:
0-100

### Философия оценки

Оценка качества приёма **НЕ ЗАВИСИТ от типа приёма** (перекус/обед/ужин). Тип —
для удобства пользователя, а не для штрафов.

Оценивается только:

1. **Состав** — БЖУ, ГИ, клетчатка, вредность
2. **Время** — штраф только за ночные приёмы (23:00-05:00)
3. **Количество** — штраф только за >800 ккал

### 🧮 Алгоритм оценки (100 баллов)

| Категория              | Max баллов | Что оценивается                   |
| ---------------------- | ---------- | --------------------------------- |
| **Калории**            | 30         | Абсолютные лимиты и время         |
| **Макросы (БЖУ)**      | 25         | Баланс белка, углеводов, жиров    |
| **Качество углеводов** | 15         | Доля простых vs сложных           |
| **Качество жиров**     | 15         | Доля полезных vs вредных + транс  |
| **ГИ и вредность**     | 15         | Средневзвешенный ГИ, индекс вреда |
| **Бонусы**             | +15        | За качественный состав            |

**Итого**: 100 base + 15 bonus = max 100 (нормализовано)

### Научная основа

Инсулиновая волна — период, когда уровень инсулина в крови повышен после приёма
пищи. В это время организм накапливает энергию, а не сжигает жир.

**Липолиз** (жиросжигание) начинается только после окончания волны.

### 🔬 Научное обоснование базовых 3 часов

**Диапазон**: 1-5 часов в зависимости від:

- GL приёма (ключевой фактор)
- Состава (жиры, белок, клетчатка)
- Индивидуальных факторов (возраст, BMI, пол)
- Времени суток

**Вывод**: Параметр `3.0` — хорошая отправная точка для смешанного приёма
средней GL. Все модификаторы корректируют это значение на ±50%.

## 🏋️ Training Context (v3.4.0)

**Файл**: `heys_iw_calc.js` (ранее `heys_insulin_wave_v1.js`) | **Функция**:
`calculateActivityContext()`

Контекст тренировки — влияние физической активности на метаболизм еды. 10 типов
контекста, которые изменяют инсулиновую волну и вредность продуктов.

### Типы контекстов

| Тип                | Бейдж               | waveBonus         | harmMultiplier | Условие                           |
| ------------------ | ------------------- | ----------------- | -------------- | --------------------------------- |
| `peri`             | 🏋️ Топливо          | до −60%           | 0.5            | Еда ВО ВРЕМЯ тренировки (±30 мин) |
| `post`             | 💪 Восстановление   | до −40%           | 0.7            | Еда ПОСЛЕ тренировки (0-6ч)       |
| `pre`              | ⚡ Перед трен.      | до −20%           | 0.6-0.8        | Еда ДО тренировки (до 90 мин)     |
| `steps`            | 👟 Активный день    | −10%              | 1.0            | >10k шагов + ужин (20:00+)        |
| `morning`          | 🌅 Утренняя трен.   | −5% (весь день)   | 1.0            | Тренировка до 10:00               |
| `double`           | 🏆 Двойная          | −10% (весь день)  | 0.9            | 2+ тренировки за день             |
| `fasted`           | 🔥 Натощак          | бонус к POST ×1.3 | 0.9            | Тренировка до первого приёма      |

## 📊 TEF & TDEE (Полные расчёты)

### TEF (Thermic Effect of Food) — Термогенез пищи

Энергия, затраченная на переваривание и усвоение пищи.

| Макронутриент | TEF (%) | Ккал/г (NET Atwater) | TEF в формуле                |
| ------------- | ------- | -------------------- | ---------------------------- |
| **Белок**     | 20-30%  | **3** (TEF встроен)  | **0** (уже учтён в 3 kcal/g) |
| **Углеводы**  | 5-10%   | 4                    | 7.5% (0.075)                 |
| **Жиры**      | 0-3%    | 9                    | 1.5% (0.015)                 |

**Формула HEYS**:

```javascript
// ВАЖНО: Protein использует NET Atwater = 3 kcal/g (уже TEF-adjusted)
// Поэтому TEF-компонент белка = 0
TEF = prot_g × 0 + carbs_g × 4 × 0.075 + fat_g × 9 × 0.015
```

> ⚠️ **Не путайте** с классическими формулами, где protein × 4 × 0.25. HEYS
> использует систему NET Atwater (protein=3 kcal/g), поэтому TEF белка уже
> встроен в калорийность. Формула `(prot×3.2 + carbs×3.5 + fat×8.8) × 0.07` —
> устаревшая и НЕ используется в коде.

### TDEE (Total Daily Energy Expenditure) — Суточный расход

```
TDEE = BMR + activityTotal + TEF + NDTE_boost
```

Где:

- **BMR** — базовый метаболизм (Mifflin-St Jeor)
- **activityTotal** — расход на активность (тренировки, NEAT, бытовая)
- **TEF** — термогенез пищи
- **NDTE_boost** — бонус от вчерашней тренировки
