# Живой справочник HEYS

Это единая точка входа в описание продукта и его реализации. Справочник не
пересказывает код: он помогает быстро найти нужную систему, понять её поток,
данные, инварианты и точки входа в реализацию, а также увидеть подтверждённые
слабые места до начала правки.

## Как устроен справочник

1. **Короткая карта проекта** — поверхности, крупные области и точки входа.
2. **Системные досье** — потоки, владельцы данных, инварианты, ошибки и ссылки
   на подтверждающий код и тесты.

Списки каждого файла и функции сюда намеренно не копируются: детали реализации
агент ищет непосредственно в актуальном коде.

## Иерархия источников

При противоречии выше стоит источник, который ближе к исполняемому поведению:

1. исходный код, схема БД и конфигурация runtime;
2. тесты и явно зафиксированные контракты;
3. документы в этом справочнике с проверенными ссылками на реализацию;
4. остальные активные документы;
5. планы, отчёты и архивы.

Большой обзор [HEYS_BRIEF.md](../HEYS_BRIEF.md) остаётся ценным продуктовым
источником, но больше не считается единственным техническим источником истины:
его факты должны подтверждаться реализацией.

## Текущий статус

Созданы короткая карта, ревизия прежних материалов, сквозное описание HEYS и
досье всех крупных областей из [матрицы покрытия](systems/README.md). Статусы
`частично` и `источник` намеренно показывают предел проверки: они не превращают
старый документ в подтверждённый контракт.

Новые досье добавляются только для самостоятельных систем, где описание
поведения действительно сокращает время разбора и предотвращает ошибки.
Изменение внутренней реализации без смены поведения, владельца, контракта или
инварианта не требует обновления справочника.

Backlog не образует третий описательный слой: это сменяемая рабочая очередь,
полученная из повторной проверки досье по коду. После исправления пункт
закрывается или удаляется из актуальной очереди, а досье хранит только
оставшийся системный риск или новый контракт.
