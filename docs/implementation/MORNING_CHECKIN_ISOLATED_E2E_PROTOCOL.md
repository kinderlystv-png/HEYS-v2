# Протокол: изолированный E2E утреннего чек-ина

## UI-гейт

Цель — один устойчивый возобновляемый flow; главное действие — сохранить текущий
шаг; слой 1 — шаг, прогресс и статус; слой 2 — тестовый trace через
`data-heys-*`; критическое не скрывать — offline, ошибку сохранения и
обязательный `YesterdayVerify`.

## Критерий успеха

- PIN-вход использует только `E2E-TestAlex`
  (`11111111-1111-1111-1111-111111111111`).
- До первого стабильного кадра завершён cloud snapshot; модалка не показывает
  пустой промежуточный шаг.
- Каждый видимый шаг сохраняется один раз, включая double tap на весе; после
  каждого шага страница закрывается и открывается снова на следующем шаге.
- `YesterdayVerify` нельзя продолжить без явного решения.
- Один шаг сохраняется offline, переживает offline-reload как `saved_local`,
  после online очередь становится пустой и статус переходит в `synced`.
- Финал и событие завершения происходят один раз; день и progress ledger
  подтверждаются отдельным cloud read-back.
- Бюджеты: стабильный кадр после sync ≤ 3000 мс; переход шага ≤ 2000 мс; offline
  resume ≤ 20000 мс.

## Изоляция и восстановление

Перед сценарием снимается полный KV snapshot разрешённого E2E-клиента. Fixture
меняет только его namespace; после теста исходные строки восстанавливаются, а
созданные тестом строки удаляются. Helper отказывается работать с UUID вне
жёсткого allow-list, поэтому пользовательский профиль не может попасть в
cleanup.

До первого анонимного boot браузеру задаётся только scoped-профиль
`E2E-TestAlex`. Это не даёт анонимному shell создать пустую pending-запись
`heys_profile` и сохраняет проверку PIN/readiness в чистом тестовом namespace.

## Протокол реализации

- [x] Безопасный exact snapshot/restore вместо timestamp-delete.
- [x] Версионированные обязательные согласия заведены как постоянная часть
      dedicated fixture.
- [x] Возраст dedicated fixture подтверждён без прохождения пользовательского
      age-gate.
- [x] Детерминированные `data-heys-step-id` / `data-heys-saving` без изменения
      пользовательского UI.
- [x] Изолированный PIN/offline/reload/cloud-read-back сценарий и бюджеты.
- [x] Фактический прогон на свежем scoped legacy-бандле и фиксация результатов.

- Пройдено 9/9 шагов; каждый подтверждён ровно один раз, включая double tap на
  весе.
- `YesterdayVerify` остался fail-closed до явного решения.
- Страница закрывалась и открывалась после каждого шага; восстановление
  продолжалось со следующего незавершённого шага.
- `sleepQuality` сохранён offline, пережил offline-reload, после возврата сети
  очередь опустела и cloud read-back подтвердил день и `synced` progress ledger.
- Первый стабильный кадр: 764 мс. Переходы шагов: 496–1230 мс. Пустых
  промежуточных кадров и повторного завершения flow нет.
- Cleanup восстановил точный исходный snapshot dedicated E2E-клиента;
  пользовательские namespace не затрагивались.
