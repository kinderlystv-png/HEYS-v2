## 3. Принципы «точно ничего не сломать»

1. **Ни одного видимого пикселя.** Эффекты не удаляем — меняем только _как_
   исполняется (`left` → `transform` с тем же таймингом) или _когда_ (пауза,
   только пока элемент/вкладка не видны).
2. **Каждая правка за LS-флагом** по существующему паттерну hot-sync
   (`heys_disable_*` / console-API). Откат = выставить флаг, без редеплоя.
3. **Не трогаем до пострелиза:** семантику interceptor-гвардов без тестов, все
   `passive:false` листенеры, blur, виртуализацию, `:has()`.
4. **Сначала измерение, потом правка.** Без INP-базлайна эффект недоказуем, а
   регресс незаметен.
5. **Скриншот-дифф** для любой правки, касающейся CSS (даже «эквивалентной»).

## 5. Чек-лист перед мержем каждой правки

1. Жесты на реальном Android: pull-refresh, свайп приёма, drag граммов,
   fingers-grid, скролл-лок открытой панели геймификации.
2. Ввод еды: добавить / изменить граммы / удалить → синк виден на втором
   клиенте; ноль новых `STALE-WRITER` warn в log-trace.
3. PIN-вход и куратор-сессия видят одинаковые продукты (инвариант №3).
4. Скриншот-дифф затронутых экранов (iPhone + Android, light/dark) — ноль
   расхождений.
5. Оффлайн-старт PWA после очистки сети.
6. INP-сценарий «добавить продукт» на P30: не хуже базлайна.

**Критерий успеха релиза:** INP на P30 в сценарии ввода < 200 мс; сценарий
«минута на вкладке Дня» — ноль long tasks > 100 мс; ноль визуальных диффов.
