## 1. Протокол отметок в мессенджере

Единая система реакций для быстрой коммуникации без лишних слов.

| Отметка | Значение             | Когда ставить                                    |
| ------- | -------------------- | ------------------------------------------------ |
| 👍      | «Вижу, принято»      | Сразу как увидел сообщение (даёт клиенту сигнал) |
| ✅      | «Внесено в дневник»  | После внесения приёма пищи                       |
| ❓      | «Нужны уточнения»    | Неясный продукт, вес, или фото нечёткое          |
| ⚠️      | «Анти-срыв / срочно» | Клиент в риске срыва, требует приоритета         |

**Правила:**

- 👍 ставим **сразу** (в рабочее время) — клиент знает, что его видят
- ✅ ставим **после** внесения — клиент знает, что дневник обновлён
- Если нужно уточнение — сначала 👍, потом вопрос с ❓

## 2. Шаблоны касаний (Pro/Pro+)

### 2.1 Утреннее касание (09:00–10:00)

**Цель:** Настроить на день, дать план.

```
Доброе утро! ☀️

Вчера: [краткий итог — норма/перебор/недобор]
Сегодня: [особенности дня — кафе/поездка/обычный]

Фокус: [1 конкретная вещь]

Удачного дня!
```

**Пример:**

```
Доброе утро! ☀️

Вчера: отлично, 1850 ккал из 2000 👍
Сегодня: обычный день, вечером ужин с друзьями

Фокус: до ужина наесть белка (творог/курица),
чтобы вечером не тянуло на хлеб

Удачного дня!
```

### 2.2 Дневное касание (14:00–15:00)

**Цель:** Корректировка по факту, что осталось на вечер.

```
Как дела?

На данный момент: [X ккал из Y]
Осталось: [Z ккал]

[Рекомендация что/когда поесть]
```

**Пример:**

```
Как дела?

На данный момент: 1100 ккал из 2000
Осталось: 900 ккал — отличный запас на ужин!

До ужина можно перекусить (орехи/фрукт),
на ужин спокойно полноценная порция.
```

### 2.3 Вечернее касание (20:00–21:00)

**Цель:** Итоги, поддержка, фиксация прогресса.

```
Итоги дня:

[Цифры: ккал, белок, общая картина]
[Что хорошо]
[Что можно улучшить — мягко]

[Поддержка/мотивация]
```

**Пример:**

```
Итоги дня:

1920 ккал — в норме! ✅
Белок 95г — супер, это важно.
Вода маловато — завтра напомню.

Хороший день! Третий подряд в норме 🔥
Спокойной ночи!
```

## 3. Сценарии Day 0 и Day 7

### 3.1 Day 0 — Первое сообщение (после регистрации)

**Цель:** Установить контакт, объяснить правила, снять тревогу.

**Когда:** В течение 30 минут после регистрации клиента.

```
Привет, [Имя]! 👋

Я [Имя куратора], буду вести твой дневник питания.

Как это работает:
1. Ты присылаешь фото еды (желательно на весах)
2. Я вношу в приложение — тебе не нужно считать
3. Вечером подвожу итог дня

Если нет весов — просто опиши словами или покажи рядом с ложкой/рукой.

Правила общения:
• Я на связи 09:00–21:00
• 👍 = вижу сообщение
• ✅ = внесено в дневник
• Если сложно/срыв — пиши, не стесняйся

Первая неделя — просто фиксируем как есть, без давления.
Готов(а) начать?
```

### 3.2 Day 7 — Недельный итог + выбор тарифа

**Цель:** Подвести итоги триала, показать прогресс, перевести в оплату.

**Когда:** На 7-й день, после последнего приёма пищи.

```
[Имя], неделя позади! 🎉

Твои результаты:
• Дней с дневником: [X из 7]
• Приёмов внесено: [Y]
• В норме дней: [Z]
• [Ключевой инсайт — что получилось/не получилось]

Что дальше?

Триал заканчивается. Варианты:

1️⃣ Base (1 990 ₽/мес)
   Приложение + 1 чек-ин в неделю

2️⃣ Pro (12 990 ₽/мес) — рекомендую
   Всё как сейчас: веду дневник + чат + созвон

3️⃣ Pro+ (19 990 ₽/мес)
   Как Pro + 7/7 без выходных + mid-week созвон

Какой вариант ближе?
```

## 4. Лимит слотов триала

### 4.1 Базовые ограничения (MVP)

| Параметр                 | Значение               | Почему                         |
| ------------------------ | ---------------------- | ------------------------------ |
| **Max активных триалов** | 3 одновременно         | Один куратор не вытянет больше |
| **Max клиентов Pro**     | 14                     | Норма загрузки из брифа        |
| **Общий потолок**        | 14 Pro + 3 триала = 17 | Операционный предел            |

### 4.2 Правила управления слотами

1. **Триал занимает слот** с момента первого ✅ (не регистрации)
2. **Неактивный триал** (0 приёмов за 48ч) — замораживается, слот освобождается
3. **Конверсия** триал→Pro — слот сохраняется
4. **Отмена** — слот освобождается сразу

### 4.3 Когда закрывать приём

```
Если занято 3 слота триала:
→ Новые заявки в waitlist с датой освобождения

Сообщение:
«Сейчас все слоты заняты. Освободится примерно [дата].
Оставь контакт — напишу как только место появится.»
```

## 5. Анти-срыв протокол

### 5.1 Признаки риска

| Сигнал                               | Действие                      |
| ------------------------------------ | ----------------------------- |
| «Хочу сладкого / тянет на вредное»   | Быстрый ответ с альтернативой |
| «Сорвался» / фото срыва              | Поддержка без осуждения       |
| Молчание >24ч после активности       | Инициативное касание          |
| Стресс/плохое настроение в сообщении | Эмпатия + план на вечер       |

### 5.2 Шаблон ответа на срыв

```
Бывает, это не конец света.

Что сейчас важно:
1. Не продолжать срыв («всё равно уже сорвался»)
2. Следующий приём — обычный, по плану
3. Завтра — обычный день, не «компенсация»

Что привело к срыву?
[Ждём ответ, чтобы понять триггер]
```

## 6. Стоп-ситуации

| Ситуация                                | Реакция                                                            |
| --------------------------------------- | ------------------------------------------------------------------ |
| Боль в груди, обморок, резкое ухудшение | «Обратись к врачу/скорой. Сопровождение на паузе до консультации.» |
| Беременность/ГВ                         | Убираем дефицит, мягкий режим, рекомендуем врача                   |
| Подозрение на РПП                       | Бережно рекомендуем специалиста; при отказе — прекращаем           |
| Запрос на лекарства/диагнозы            | «Это вопрос к врачу, я помогаю с питанием/режимом»                 |

## 7. Короткие объяснения клиенту (согласовано с приложением)

Используйте те же формулировки, что заложены в HEYS, чтобы не спорить с
продуктом и калькулятором.

- **Дефицит** — это снижение целевой калорийности относительно нормы дня
  (расхода). Устойчивый дефицит обычно переносится легче, чем экстремальный; при
  затяжном жёстком режиме возможна метаболическая адаптация (в приложении —
  инсайт «Адаптивный термогенез»).
- **Вес на весах** — дневные колебания частично связаны с водой и гликогеном, не
  только с жиром; ориентир — недельный тренд (инсайт «Прогноз веса»; в профиле
  расчёт цели — Hall и др.).
- **Шаги и NEAT** — дополняют расход и переносимость режима; основной рычаг
  энергетического баланса — питание относительно нормы дня.
- **Тренировки при дефиците** — помогают сохранять мышечную массу и силовые
  показатели; учёт в приложении — калории и самочувствие. Питание остаётся
  главным рычагом дефицита.

Медицинские вопросы, боль, беременность, РПП — по протоколу раздела 6.
