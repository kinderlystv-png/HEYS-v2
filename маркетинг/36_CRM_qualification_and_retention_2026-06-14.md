# 36 · CRM qualification and retention — 2026-06-14

> **Статус:** evidence/reference. Это правила, подготовленные конкретным pass;
> живые CRM/retention-задачи и статусы ведутся в `22`.

Источник для закрытия строк `22` §3.3 и §3.4. Документ фиксирует правила,
которые уже можно применять в Ф0 без новой CRM-системы: bot handoff создаёт
`lead_id`, `source`, `segment`, `readiness` и безопасный summary, а куратор
ведёт решение по таблице ниже.

## 1. Lead handoff

Минимальный набор полей для каждой заявки:

| Поле                  | Откуда берётся                                       | Зачем куратору                                |
| --------------------- | ---------------------------------------------------- | --------------------------------------------- |
| `lead_id`             | `public.leads.id`                                    | Ссылка на заявку без пересылки телефона в чат |
| `source` / `campaign` | `/start` payload или organic                         | Понять канал и связать с недельной воронкой   |
| `segment`             | bot quiz: `chaotic`, `evening`, `emotional`, `mixed` | Первый контекст разговора                     |
| `readiness`           | `this_week` / `next_week`                            | Когда предлагать старт                        |
| `goal`                | quiz summary                                         | Что человек считает главной задачей           |
| `barrier`             | quiz summary                                         | Что мешало раньше                             |
| `frequency`           | quiz summary                                         | Насколько хаотичен режим                      |
| `past_method`         | quiz summary                                         | Какие подходы уже пробовали                   |

Правило приватности: телефон и имя хранятся в `public.leads`, но не уходят в
кураторский Telegram handoff. В чат уходит только `lead_id` и summary.

## 2. Qualification rules

| Условие                              | Квалификация          | Следующий шаг                                         |
| ------------------------------------ | --------------------- | ----------------------------------------------------- |
| `readiness=this_week` + есть телефон | Hot                   | Ответить в тот же день, предложить слот первой недели |
| `readiness=next_week`                | Warm                  | Согласовать дату старта, напомнить за 1-2 дня         |
| `segment=chaotic`                    | Pro-first             | Объяснить ценность дневника и режима недели           |
| `segment=evening`                    | Pro-first             | Начать с вечерних решений, сна и распределения еды    |
| `segment=emotional`                  | Pro / Pro+ по ёмкости | Сначала снизить риск срыва, не давить целью веса      |
| Нет телефона после `week_request`    | Incomplete            | Один мягкий follow-up в Telegram, затем без давления  |

Нельзя предлагать квартальную или годовую предоплату до готовой оферты,
pricing-решения и биллинга. В Ф0 продаём только текущий согласованный продукт и
ручной платёж.

## 3. Curator first-contact script

1. Назвать источник заявки: «Вижу заявку на бесплатную неделю HEYS».
2. Коротко отзеркалить паттерн: «По ответам похоже, что сейчас сильнее всего
   мешает вечерний/хаотичный/эмоциональный сценарий».
3. Снять ожидание: «На первой неделе мы не обещаем быстрый результат. Сначала
   собираем дневник и находим один устойчивый шаг».
4. Предложить слот старта: «Можем начать на этой неделе или перенести на
   следующую?»

## 4. Win-back triggers

| Момент       | Триггер                         | Действие                                                            |
| ------------ | ------------------------------- | ------------------------------------------------------------------- |
| День 1       | Человек не начал дневник        | Напомнить, что можно прислать первый приём еды в любом виде         |
| День 2       | Есть дневник, но мало контекста | Попросить один недостающий факт: сон, время еды или настроение      |
| День 4-5     | Был срыв / пропуск              | Вернуть без стыда: «пропуск не ломает неделю, важен следующий день» |
| Конец недели | Нет решения по оплате           | Показать 1-2 факта недели и предложить следующий конкретный шаг     |
| После отказа | Причина известна                | Отложить контакт на 14-21 день, если человек сам согласился         |

Win-back не должен спорить с отказом. Если человек явно просит не писать,
контакт останавливается и фиксируется в CRM-заметке.

## 5. Review checklist

- `lead_id`, source, segment and readiness are visible in curator handoff.
- Phone/name are not duplicated into Telegram handoff.
- Weekly funnel report can aggregate `lead`, `quiz_start`, `quiz_complete`,
  `week_request`, `trial_active`, `payment`, `renewal`.
- Curator scripts avoid medical promises, weight guarantees and pressure.
