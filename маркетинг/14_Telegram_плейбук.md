# HEYS — Telegram: продвижение, ведение, темы (плейбук)

> Практический how-to по Telegram. Обновлено: 2026-06-16. Telegram — ключевой
> канал для РФ-посевов и инфлюенс-трафика; рыночные оценки на 2026 год дают
> широкий коридор, поэтому все бюджеты и CPM ниже — ориентиры для теста, а не
> норматив. Тон всего контента — строго по `apps/landing/COPY_VOICE.md`:
> серьёзный, спокойный, доказательный; без слэнга, эмодзи-спама, «-N кг»,
> «24/7», «психолог», «вылечу».

## 0. Два разных Telegram — не путать

|               | Публичный канал бренда               | Личная переписка куратор ↔ клиент |
| ------------- | ------------------------------------ | ---------------------------------- |
| Зачем         | Маркетинг: прогрев, заявки на неделю | Операционка: ведение, анти-срыв    |
| Этот документ | **да**                               | нет (см. бриф/SLA)                 |

Дальше — только про **публичный маркетинговый канал** и воронку из него.

## 1. Роль Telegram в воронке HEYS

- Ключевой **органический + посевной** канал прогрева к бесплатной неделе.
- Длинная тёплая аудитория, возможность тестировать темы до платных посевов.
- ⚠️ В посевах высокий риск накруток и нестабильной статистики → любые закупки
  только с проверкой TGStat/Telemetr и тестовой UTM. Меряем CPL и заявки, а не
  «охваты».
- Связка: **канал (прогрев) → лид-магнит-бот → заявка на неделю → куратор.**
- На старте канал не должен выглядеть как медиа “про питание вообще”. Его задача
  уже с первого экрана вести к персональному разбору: срывы → паттерн недели →
  сопровождение → бесплатная неделя → ограниченная ёмкость.

## 2. Настройка канала (чек-лист)

- [ ] **Имя и @username** — простые, по бренду (HEYS), легко искать/диктовать.
- [ ] **Аватар** — чистый логотип, премиум, без коллажей.
- [ ] **Описание** — одна строка позиционирования: «Сопровождение питания с
      живым куратором. Вас ведут, а не оставляют с цифрами».
- [ ] **Закреплённый пост** — что это + кому + оффер (бесплатная неделя, без
      карты) + кнопка/ссылка на бот-магнит. Обновлять по мере набора («осталось
      N мест»).
- [ ] **Комментарии включены** — обсуждение поднимает ER и доверие.
- [ ] **Навигация** — хэштеги-рубрики (#срывы #какработает #кейсы #честно) или
      меню в закрепе.
- [ ] **Дисклеймер** — «не медицинская услуга» там, где уместно.

### 2.1 Стартовый набор канала

| Элемент         | Что должно быть                                                                                 |
| --------------- | ----------------------------------------------------------------------------------------------- |
| Аватар          | Логотип HEYS, без фото еды/веса/коллажей                                                        |
| Описание        | Одна фраза: сопровождение питания с куратором, без медицинских обещаний                         |
| Закреп          | Оффер первого набора + ссылка на бот/форму + ограничение по слотам; публиковать только после R0 |
| Первые 6 постов | 2 про механику срывов, 2 про кухню сопровождения, 1 про анти-марафон, 1 честный FAQ             |
| Ссылки          | Все ссылки с UTM/промокодом после R0; до R0 — без формы/бота/сбора ПДн                          |

До R0 канал ведётся как контентный: без CTA на форму/бот и без сбора телефона,
ФИО или health-данных в комментариях/личных сообщениях. После R0 и запуска бота
закреп ведёт в «Твой тип срыва»; форма остаётся резервом.

## 3. Контент-система: рубрики (повторяемый каркас)

Не «придумывать каждый раз», а крутить 5–6 рубрик по воронке:

| Рубрика                      | Цель в воронке                                   | Частота |
| ---------------------------- | ------------------------------------------------ | ------- |
| **Механика срывов**          | Узнавание себя + снятие вины                     | 1/нед   |
| **Анти-марафон / отстройка** | Отличие от дешёвых решений                       | 1–2/мес |
| **Кухня сопровождения**      | Показать ценность куратора (без overpromise)     | 1/нед   |
| **Кейсы**                    | Доверие через процесс, не через килограммы       | 2–3/мес |
| **Прозрачность**             | Честный SLA, «не медуслуга», где мы НЕ всесильны | 1–2/мес |
| **Польза/режим**             | Сохраняемые опоры для реальной недели            | 1/нед   |

**Форматы (чередовать):** текст-пост (основа), лонгрид (Telegraph/статья),
голосовое (доверие к эксперту), видео-кружок, опрос/квиз (ER + квалификация),
карусель «до/после процесса». CTA на неделю — только после R0, мягкий, в 1 из
3–4 постов.

**Структура поста:** крючок в первой строке (её видно в превью) → одна мысль →
вывод/что делать → мягкий CTA (не в каждом).

## 4. Ритм и ведение

- **3–4 поста/нед + 1 лонгрид/нед** на старте. Регулярность важнее объёма; не
  гнать ежедневно ради галочки.
- **Контент-конвейер:** 1 день в неделю — батч (пишешь на неделю вперёд из банка
  тем), ставишь в отложенный постинг. Это экономит bandwidth (см. док 10).
- **Время постинга (РФ-ориентир):** утро 8–10 и вечер 19–22; уточнять по своей
  аудитории (статистика канала).
- **Engagement:** первые 30–60 минут после поста — отвечать в комментариях,
  держать обсуждение; реакции и опросы повышают охват.

### 4.1 Первые 14 дней

| День | Действие                                                                                                   |
| ---- | ---------------------------------------------------------------------------------------------------------- |
| 0    | Канал, аватар, описание; закреп и форма/бот-ссылка — только после R0                                       |
| 1    | П1 из `24`: вечерние срывы как физиология и контекст                                                       |
| 3    | П4 из `24`: как выглядит сопровождение и что делает куратор                                                |
| 5    | П7 из `24`: где мы не всесильны; дисклеймер «не медицинская услуга»                                        |
| 7    | Опрос П9 без давления: когда чаще тянет сорваться                                                          |
| 9    | П3 из `24`: анти-марафон, почему жёсткие режимы не держатся                                                |
| 11   | П5 из `24`: что делает куратор, когда видит риск срыва                                                     |
| 13   | После R0: мягкий CTA на бесплатную неделю без карты, количество мест ограничено реальной ёмкостью куратора |

Цель первых 14 дней после R0 — не “набрать медиа”, а проверить: люди переходят в
заявку/бот, задают вопросы, узнают себя в боли срывов.

## 5. Банк тем (готовые идеи постов)

**Механика срывов**

1. «Почему вечером рука сама тянется к холодильнику» (физиология, не слабость
   воли).
2. «3 признака, что вы на грани срыва — за 2 дня до него».
3. «Срыв — это не провал. Что происходит на самом деле и почему это нормально».
4. «Усталость, недосып, стресс → переедание: как это связано».
5. «Почему “с понедельника” не работает».

**Анти-марафон / отстройка** 6. «Почему жёсткие марафоны не держатся после
первого рывка». 7. «Чем системное сопровождение отличается от марафона». 8.
«Трекер посчитает калории, но не напишет вам в момент срыва». 9. «AI-приложение
vs человек рядом: в чём разница для результата».

**Кухня сопровождения** 10. «Как выглядит день с куратором» (без
преувеличений). 11. «Что делает куратор, когда видит риск срыва». 12. «Дневник
за вас: как это занимает 5 секунд». 13. «Почему мы вносим еду руками, а не
отдаём всё алгоритму». 14. «Недельный созвон: что на нём происходит».

**Кейсы (процесс, не цифры)** 15. «Как Х перестал срываться по вечерам» (с
согласия, без веса). 16. «Возвращение после двух недель “всё пропало”». 17. «Как
клиент перестал считать калории и стал спокойнее».

**Прозрачность / доверие** 18. «Наш честный SLA и где мы НЕ всесильны». 19.
«Почему мы берём ограниченное число людей». 20. «Это не медицинская услуга — что
это значит». 21. «Как мы храним ваши данные (в России)».

**Польза / режим** 22. «Что съесть в кафе, чтобы не выпасть из режима». 23.
«Поездка/праздник без срыва: 3 простых правила». 24. «Завтрак, который держит до
обеда». 25. «Почему “правильный” ужин важнее “правильного” завтрака». 26. «Вода
и режим сна — недооценённые рычаги».

**Вовлечение/квизы** 27. Опрос: «Когда вас чаще всего тянет сорваться?»
(утро/день/вечер/ночь). 28. Квиз: «Твой тип срыва» → ведёт на бот-магнит. 29.
«Закончите фразу: я срываюсь, когда…».

**Сезонное** (см. сезонность в доке 10) 30. Январь: «Как не слить новогодние
обещания к февралю». 31. Март–май: «“К лету” без насилия над собой». 32.
Сентябрь: «Возвращение в режим после лета».

## 6. Продвижение

### 6.1 Органика (бесплатно)

- Лид-магнит в закрепе → бот.
- Взаимопиар/коллаборации с **непрямыми** (фитнес-тренеры, ЗОЖ-каналы, темы
  заботы о себе/выгорания) — не конкуренты, общая аудитория.
- Репосты в смежные чаты **с разрешения**, не спам.
- UGC и отзывы клиентов (с согласия).
- Ключевые слова в первых строках поста — TG-поиск их индексирует.

**Порядок органики:** сначала личный круг и тёплые репосты → затем взаимопиар с
смежными каналами → только после первых заявок тестовые посевы. Не начинать с
покупки охватов: пока нет факта лид→триал, платный трафик может просто сжечь
слоты куратора.

### 6.2 Посевы (платные размещения в чужих каналах)

Критерии отбора по **TGStat** (иначе деньги в накрутку):

- Тематика: забота о себе, ЗОЖ, осознанность, мам-сообщества, выгорание — **не**
  «худеем к лету».
- ER ≥ 1,5–2%; охваты стабильны (без «пилы»).
- **Доля ботов/накруток < 10%.**
- Органический прирост (без всплесков от giveaway).
- Тест: 1 интеграция → отдельный промокод/UTM → замер заявок → решение о
  повторе.

**Шаблон запроса (outreach):**

> Здравствуйте! Мы — сервис сопровождения питания HEYS. Хотим разместить честный
> экспертный пост о том, почему срывы — это не слабость воли, со ссылкой на
> бесплатную неделю. Подскажите стоимость, формат и свежую статистику канала
> (охваты, ER). Размещение оформим с маркировкой (ERID).

### 6.3 Telegram Ads

- Вход через **eLama** — на июнь 2026 в публичных материалах заявлены запуск с 0
  € для юрлиц/ИП, комиссия 15% по акции и оплата по факту расходов; условия
  перепроверять перед запуском.
- Таргет: по тематическим каналам-конкурентам (марафоны/ЗОЖ) и по интересам.
- CPM и входной бюджет зависят от кабинета, реселлера, темы и конкуренции;
  планировать тест от CPL, а не от охватов.
- Креатив: короткий хук + ссылка на **канал/лид-магнит**, не сразу на оплату.

### 6.4 Чего НЕ делать

Не покупать накрученные каналы; не гнать холодный трафик сразу в оплату; не
спамить в чаты без разрешения; не использовать запрещённые формулировки.

## 7. Воронка: Telegram → бесплатная неделя

```
Холодный (Ads / посев)  →  подписка на канал ИЛИ сразу бот-магнит
Тёплый (контент канала) →  прогрев  →  заявка на неделю
        ↓
Бот: лид-магнит («Карта твоих срывов» / квиз) → контакт → квалификация
        ↓  запись на неделю = по ёмкости куратора (не больше!)
Передача куратору → активный триал → оплата
```

- Лид-магниты: тест «твой тип срыва», чек-лист «карта триггеров», 1 бесплатный
  разбор фото еды.
- Все ссылки — с **UTM/промокодом** (аналитика намеренно урезана под 152-ФЗ, см.
  док 10; без меток мерить нечем).
- Поток заявок не превышает ёмкость кураторов — иначе список ожидания, не
  сорванный SLA.

## 8. Метрики (TGStat + свой учёт)

| Метрика                             | Ориентир                                                                                 |
| ----------------------------------- | ---------------------------------------------------------------------------------------- |
| Прирост подписчиков                 | растёт органически                                                                       |
| ER (вовлечённость)                  | ≥ 1,5–2%                                                                                 |
| Охват поста / подписчики            | стабилен                                                                                 |
| Доля ботов (в посеве-доноре)        | < 10% (иначе не повторяем)                                                               |
| Переходы в бот (CTR закрепа/поста)  | мерить, растить                                                                          |
| CPL из Telegram                     | под целевым (CAC ÷ лиды-на-оплату)                                                       |
| Лид → неделя → оплата (источник TG) | базовый ориентир: лид→неделя ≥40%, неделя→оплата ≥30%; уточнить после первых 30–50 лидов |

## 9. Легал (ERID + 152-ФЗ)

- **Свой контент в своём канале** про свои услуги — реклама не нужна, ERID не
  требуется.
- **Посевы и Telegram Ads** — ERID обязателен: через eLama/ОРД-процесс или
  вручную по `33`, ERID **до публикации**, договор с пунктом о маркировке, отчёт
  в ОРД не позднее 30 календарных дней после месяца размещения. 3% рекламный
  сбор проверять договорно по роли площадки/исполнителя, потому что платит не
  всегда рекламодатель напрямую.
- **152-ФЗ:** данные лидов через бот — минимизировать передачу на сторонние
  сервисы (по брифу — только `lead_id`, без ПДн); хранение в РФ.
- **РКН-уведомление оператора ПДн:** до активного сбора заявок через Telegram
  закрыть задачу `22` 1.7 — ИП подаёт уведомление на портале персональных данных
  Роскомнадзора, потому что канал/бот/форма собирают контактные данные и данные
  о сервисном сопровождении.
- **Формулировки:** без «24/7», «гарантированно похудеете», «психолог»,
  «вылечу».

## 11. Библиотека рекламных примеров

Цель раздела — не копировать чужие креативы, а собирать паттерны: какой крючок
использован, почему он сработал/не сработал и как безопасно адаптировать вывод
под HEYS.

### Как описывать новый скрин

1. Дать ID: `TG-AD-00N`.
2. Указать источник/формат без лишних персональных данных.
3. Коротко назвать оффер и аудиторию.
4. Разобрать: что работает, что слабое, что взять для HEYS, чего избегать.
5. Обязательно добавить `Вывод для HEYS`: один короткий абзац, что именно
   переносим в нашу систему и что не копируем.
6. Поставить оценки 1–5: хук, доверие, визуал, CTA, соответствие бренду.
7. Если это платный креатив для нас — отдельно проверить ERID/договор/ОРД по
   `33` до публикации.

### TG-AD-001 — Фитнес / похудение / нативная карточка под постом

Скрин: assets/telegram-ads/TG-AD-001.jpg  
Источник: Telegram-канал, скрин от пользователя  
Дата фиксации: 2026-06-16  
Формат: рекламная карточка под постом  
Цель рекламы: переход в публикацию / канал  
Аудитория: мужчины, интерес к форме тела / похудению  
Оффер: убрать живот через разбор ошибок в питании  
Паттерн: боль → снятие вины → конкретная причина → переход к разбору  
Оценки: хук 4/5; доверие 2/5; визуал 2/5; CTA 4/5; fit HEYS 3/5 Вывод для HEYS:
взять структуру «боль → не слабость → есть причина → разбор», но заменить тему
живота и давления на внешность на контекст срывов, усталости, режима и
сопровождения; визуальный тон сделать спокойнее и доказательнее.

Что работает:

- Крупный простой заголовок на картинке сразу называет боль.
- Фраза «сила воли тут ни при чем» снимает вину и цепляет внимание.
- Есть конкретика: не просто мотивация, а ошибки в питании.
- CTA ведёт не сразу в покупку, а в публикацию с разбором.

Что слабое:

- Визуал выглядит шаблонно и близко к инфобизнес-рекламе.
- Недостаточно доверия: непонятно, кто эксперт и почему ему верить.
- Хук на грани кликбейта; для HEYS такой тон лучше смягчать.
- Узкий акцент на «живот» может увести в обещания быстрого похудения.

Что взять для HEYS:

- Структуру «боль → это не слабость → есть понятная причина → разбор внутри».
- Конкретный обещанный разбор вместо широкого обещания результата.
- Мягкий переход в полезный материал / лид-магнит, а не сразу в оплату.
- Формулировку вокруг срывов, усталости, режима и сопровождения, а не вокруг

### Построение одного поста

1. Ситуация: узнаваемый фрагмент недели, без стыда и давления.
2. Наблюдение: почему это повторяется, какой контекст обычно влияет.
3. Разбор: один фактор, один практический вывод, без перегруза.
4. Опора: чек-лист, мини-карта, сравнение или вопрос для самопроверки.
5. CTA: квиз, карта срывов или Pro-неделя; не прямая продажа.

### Что должно повторяться постоянно

- Главная рамка: срыв не равен слабости; чаще это повторяющийся паттерн недели.
- Роль HEYS: клиент присылает данные, куратор ведёт дневник и видит картину
  недели.
- Продуктовый мост: общий пост помогает понять тему, квиз или Pro-неделя дают
  персональный разбор.
- Доверие: спокойный тон, реальные ограничения, без обещаний результата и
  медицинских заявлений.
- Ритм: 3 полезных поста на 1 мягкий переход в квиз или заявку.

### Чего не делать

- Не превращать канал в рецепты, спортпит, фитнес-мотивацию или научпоп обо всём
  организме.
- Не копировать чужую агрессию, спорность, скидочные механики и «до/после».
- Не обещать похудение, сроки, отсутствие срывов или медицинский эффект.
- Не делать CTA только «пишите в личные сообщения»: основной путь должен быть
  квиз, заявка или бесплатная Pro-неделя.
- Не писать посты без продуктового моста: каждый материал должен объяснять,
  зачем нужен куратор, дневник или персональный разбор.
