## Как читать

1. Начни с `01` — контекст и факты рынка.
2. `02` — где мы сильны/слабы (там же 2 блокера запуска).
3. `03` + `04` — что и как делаем (стратегия).
4. `06–09` — операционная глубина: конкуренты, плейбуки каналов,
   воронка/онбординг, supply кураторов.
5. `10` — обязательно прочитать перед запуском: что мы могли упустить и что
   делать СЕЙЧАС (легал, оплата, измеримость, этика).
6. `11` — фазовая модель цены: как запускаться первым набором и когда поднимать
   цену.
7. `19` — текущий канон Фазы 0: Self / Pro / Pro+, триал после квалификации,
   Lite открыт, Solo AI позже.
8. `20` — текущая AI-рамка: технически возможно, но в Pro не используем;
   вернуться после ручной Фазы 0.
9. `21` — позиция по данным: не обещать полуработающий AI, строить маркетинг
   вокруг чистых данных.
10. `05` — экономика: «Фазы-цена» (старт vs масштаб), «Реальность-чек» (честный
    LTV:CAC ≈2,7x), «Сценарии»/«Чувствительность» для устойчивости.

## Приоритеты NOW

1. **Запуск Фазы 0 = Self / Pro / Pro+** — Self 490 ₽ как самостоятельный
   урезанный режим; Pro 7 990 ₽ для первого набора, 7 дней только после
   квалификации; Pro+ индивидуально (`19`). Lite — открытый вопрос, Solo AI —
   позже.
2. **AI-распознавание** — не обещать и не внедрять в Pro; ручной ввод куратора
   нужен для внимания, опыта и будущей методологии (`20`).
3. **Позиция по данным** — вынести в лендинг/FAQ: HEYS не заменяет точность
   полуработающей автоматизацией (`21`).
4. **Юр-структура кураторов (НПД)** — готовить заранее: риск переквалификации; с
   2026 работа с >35 НПД = индикатор проверки. Договоры на результат, не на
   процесс.
5. **Оплата Pro** — ручная помесячная; продление предлагает куратор, когда видит
   позитивный момент и понимает контекст клиента.
6. **Измеримость при отключённой аналитике** — промокоды + «откуда узнали» +
   серверная Metrica.
7. **ERID-маркировка + сбор 3%** — до первой закупки платной рекламы (в Фазе 0
   трафик в основном тёплый/органический).

> **Про «блокер W1» (экономика куратора).** На Фазе 0 он не действует в полном
> виде: первый набор идёт на внутреннем ресурсе, без внешней выплаты куратору по
> полной ставке. Блокер превращается в задачу Фазы 1: на сколько поднять цену
> под зарплату внешнего куратора (лист «Фазы-цена», ориентир ~9 900–10 900 ₽).
> Старая цифра «фикс 7 500» из брифа — под устаревшую цену 12 990.

## Важная оговорка по цифрам

Цены и операционные параметры (тарифы, загрузка кураторов) — из проекта
(`pricing.ts`, `docs/HEYS_BRIEF.md`). Маркетинговые допущения (CAC, конверсии,
отток, комиссии) — рыночные ориентиры/бенчмарки, помеченные в финмодели как
ДОПУЩЕНИЯ. Их нужно заменить фактом по мере накопления данных за Фазу 1–2.

## Ритм актуализации (cadence)

Чтобы корпус не устаревал и не расползался — лёгкий еженедельный ритуал, **≤
15–30 мин**:

1. **KPI-факт** — свежие цифры в лист «KPI-трекер» `00_Сводная_панель.xlsx`;
   проверить красные метрики.
2. **Статусы** — обновить `22_План` и `25_Roadmap` (что сдвинулось, что
   красное).
3. **Решения** — договорённости недели → в журнал `15_Ревизия` §2 (а не в чат).
4. **Datestamp-проверки** — освежить при устаревании: CalZen-цены (`12`/`26`),
   закон о рекламе нутрициологии (`10`/`22` п.2.5).

Правила:

- **Триггер changelog:** правка ключевого дока > 10% или изменение канона →
  запись в Журнал решений `15` §2 с датой. Отдельный `CHANGELOG.md` не заводим.
- **Вторая пара глаз:** правки канона и клиент-видимых текстов (лендинг, посты,
  оффер) проходят ревью (практика реализатор↔ревьюер).
- **Без новых файлов ради процесса** — cadence живёт в README и `15`, не в
  отдельном доке.
- **Реклама нутрициологии:** позиция «сопровождение, не медуслуга» — канон (`10`
  §3.2 / `COPY_VOICE`); статус законопроекта проверяется в еженедельном
  datestamp-чеке (п.4).

## Quick-start (новому участнику / себе после паузы)

Путь чтения за ~30 мин:

1. `15_Ревизия` — текущая картина + журнал решений (источник истины).
2. Канон: `19` (Pro-first), `20` (AI-рамка), `21` (данные важнее AI).
3. Этот `README` — карта пакета и навигаторы (`26` — конкурентка).
4. `22_План` + `25_Roadmap` — что делать сейчас, гейты и красные триггеры.
5. Брать незакрытую (⬜) задачу из `22`.

> **Bandwidth основателя — узкое место.** К дню ~30 Фазы 0
> делегировать/аутсорсить операционку набора и контент-постинг (снимается с
> основателя первым; ведение клиентов — позже). См. `25` § Bus-factor.
