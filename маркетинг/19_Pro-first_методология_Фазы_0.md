# HEYS — Pro-first методология Фазы 0

> **Статус:** canonical reference. Канон запуска Фазы 0; живые задачи и статусы
> ведутся в `22`.

> Решение от 2026-06-09. На первом этапе не продаём AI-лестницу и не строим
> тарифы вокруг распознавания еды. Рабочая сетка запуска: Self как дешёвый
> самостоятельный режим, Pro как главный продукт, Pro+ как премиум-надстройка.

---

## 1. Канон Фазы 0

**Главный оффер:** Pro для первого набора из 8–12 активных клиентов.

**Бюджетный якорь:** Self — самостоятельный режим с урезанным функционалом, без
куратора и без сильных модулей HEYS.

| Параметр        | Решение Фазы 0                                                       |
| --------------- | -------------------------------------------------------------------- |
| Главная продажа | Pro-сопровождение с куратором                                        |
| Сетка запуска   | Self / Pro / Pro+                                                    |
| Self            | 490 ₽/мес; самостоятельный дневник и базовый контроль                |
| Pro             | 7 990 ₽/мес для первого набора; дальше цена выше                     |
| Pro+            | 14 990 ₽/мес; индивидуально, если нужен повышенный уровень внимания  |
| Вход            | заявка → квалификация → 7 дней старта → оплата                       |
| Ёмкость         | не больше 8–12 активных одновременно, чтобы остались часы на плейбук |
| Триал           | бесплатные 7 дней только после квалификации                          |
| Суть Pro        | клиент присылает факты, куратор сам заносит питание/тренировки       |
| Lite            | не тариф запуска; открытый вопрос на будущее                         |
| Solo AI         | не нужен для запуска; вернуться после ручной Фазы 0 как R&D          |
| Главный KPI     | доказать, что Pro покупают, проходят первый месяц и продлевают       |

**Почему так:** Self даёт имитацию выбора и широкий вход, но не конкурирует с
Pro, потому что не содержит главную ценность HEYS: человека, который ведёт
контекст и снимает рутину. AI не нужен как условие запуска Self.

**Позиция по данным:** HEYS не продаёт полуработающий AI как фичу ради эффекта.
В Self ценность строится на самостоятельном дневнике и личной базе продуктов. В
Pro/Pro+ — на том, что куратор аккуратно переносит присланные данные и остаётся
в контексте клиента. Честная точность важнее эффектной автоматизации.

## 2. Сетка запуска

| Тариф |                     Цена | Что продаём                                                                      | Чего нет                       |
| ----- | -----------------------: | -------------------------------------------------------------------------------- | ------------------------------ |
| Free  |                      0 ₽ | квиз, демо, контент, заявка на квалификацию                                      | личного кабинета как ценности  |
| Self  |                490 ₽/мес | самостоятельный дневник, КБЖУ/приёмы, базовая динамика, виджеты, задачник        | куратора, инсайтов, анти-срыва |
| Pro   | 7 990 ₽/мес первый набор | куратор ведёт дневник по присланным данным, включает аналитику и держит контекст | автосписания, AI-обещания      |
| Pro+  |             14 990 ₽/мес | Pro + приоритет, тренировки глубже, больше ручного внимания                      | массовой доступности           |

Self должен быть достаточно полезным, чтобы его не стыдно продавать, но
достаточно ограниченным, чтобы человек ясно видел разницу: “сам” против “за меня
ведёт куратор”.

### Self: что включено

- самостоятельное ведение дневника питания;
- базовые расчёты КБЖУ и приёмов пищи;
- базовая динамика по дням/неделе;
- виджеты/навигатор;
- задачник;
- ручной ввод продуктов и тренировок;
- сохранение истории.

### Self: что не включено

- чат с куратором;
- ревью куратора;
- перенос данных куратором;
- AI-распознавание еды;
- инсайты и интерпретация причин;
- анти-срыв и инициативные сообщения;
- приоритетная поддержка.

## 3. Что именно проверяем

Фаза 0 отвечает не на вопрос “можно ли привлечь дешёвый лид”, а на вопрос:
**готов ли человек платить за внимательную внешнюю опору, порядок недели и
сопровождение, которое помогает вернуться без стыда.**

Проверяем 5 гипотез:

1. **Ценность.** Клиент чувствует пользу уже за первую неделю: меньше тревоги,
   больше ясности, понятен следующий шаг.
2. **Цена.** 7 990 ₽ не воспринимаются как “дорогой счётчик”, если продана
   связка “приложение + куратор, который ведёт дневник за клиента + держит
   контекст недели + помогает вернуться после отката”.
3. **Операционка.** Куратор может вести 8–12 клиентов без потери качества, а
   команда параллельно собирает плейбук.
4. **Конверсия.** Квалифицированный триал конвертируется в оплату заметно лучше,
   чем широкий бесплатный поток.
5. **Повторяемость.** То, что делает куратор руками, можно описать так, чтобы
   позже передать внешнему куратору.
6. **Self как якорь.** Self покупают как дешёвый самостоятельный режим, но он не
   забирает мотивацию к Pro, а показывает разницу “сам” против “с куратором”.

## 4. Что обещаем в маркетинге

Внешнее обещание должно быть уже, чем весь продуктовый потенциал.

**Говорим:**

- HEYS помогает вернуть порядок без жёстких диет и одиночной рутины.
- Если нужен только дневник и базовый контроль — есть Self.
- В Self Вы ведёте дневник и постепенно формируете свою базу продуктов.
- В Pro клиент не обязан сам заполнять дневник: он присылает еду и тренировки, а
  куратор аккуратно переносит данные в HEYS, чтобы аналитика работала вовремя.
- HEYS строит рекомендации на данных, которым можно доверять.
- Мы не заменяем точность красивой автоматизацией: если функция не даёт
  надёжного результата, мы не делаем её главным обещанием.
- У Вас есть приложение для дневника и куратор, который ведёт данные, смотрит на
  ситуацию, держит контекст недели и пишет в рабочем процессе, когда видит риск
  срыва.
- Первая неделя нужна, чтобы понять Ваш контекст и собрать рабочий план.
- Первый набор идёт со скидкой; дальше Pro будет дороже.

**Не говорим на Фазе 0:**

- “AI сам распознаёт еду” — пока это не продуктовый факт.
- “AI точно считает еду по фото” — даже после MVP это должно быть черновиком,
  который проверяет человек или сам клиент.
- “Гарантированно идеально” — overpromise. Правильно: куратор аккуратно
  переносит данные и отвечает за чистоту дневника в рамках сервиса.
- “Self заменяет Pro” — нет. Self = самостоятельный режим; Pro = HEYS с живым
  человеком в контуре.
- “Куратор берёт на себя всю работу вообще” — overpromise. Правильно: куратор
  берёт на себя рутину дневника и внесения данных, а клиент присылает факты и
  остаётся участником процесса.
- “Точная математика сжигания жира” — звучит как медицинское/научное обещание,
  которое сложнее доказать.
- “Выберите один из пяти тарифов” — на старте это лишняя сложность.

## 5. Воронка Фазы 0

```text
Тёплый контакт / контент / личная рекомендация
→ заявка
→ короткая квалификация
→ слот на 7 дней
→ активный триал
→ разбор результата недели
→ Pro 7 990 ₽
→ 30 дней сопровождения
→ продление / кейс / отзыв
```

Self живёт отдельной короткой веткой:

```text
Лендинг / демо
→ Self 490 ₽
→ самостоятельное ведение
→ ограничения функционала
→ апгрейд в Pro, если человеку нужна помощь и снятие рутины
```

### Квалификация заявки

Цель квалификации — не “отсечь людей”, а защитить качество сопровождения.

Минимальные вопросы:

- Что сейчас не получается удержать?
- Какой результат человек хочет получить за ближайший месяц?
- Готов ли он утром делать короткий чек-ин и присылать питание/тренировки 7
  дней?
- Есть ли ограничения, при которых нужен врач, а не HEYS?
- Готов ли он начать в ближайшую неделю?

Если человек не готов начинать, лучше переводить в контент/лист ожидания, чем
занимать слот триала.

### Протокол данных для Pro

От клиента требуется не самостоятельное ведение дневника, а регулярная передача
фактов:

- утренний чек-ин: сон, самочувствие, вес/объёмы при необходимости, планы на
  день;
- питание в течение дня: что съел, продукт, граммы или понятная оценка порции,
  фото/сообщение в мессенджере;
- тренировки: тип, длительность, пульсовые зоны или средний/максимальный пульс,
  если есть;
- уточнения по запросу куратора, если данных не хватает для корректного ввода.

Куратор переносит это в HEYS, уточняет спорные места и следит, чтобы аналитика
оставалась чистой и своевременной. Предпочтительный канал — встроенный
мессенджер HEYS; до его полного внедрения допустим внешний мессенджер как
операционный костыль.

## 6. Lite — открытый вопрос

Lite пока не должен конкурировать ни с Self, ни с Pro. Сейчас непонятно, что это
за продукт и какую работу он делает в модели.

Гипотезы, которые можно проверить позже:

- сохранить клиента, которому Pro дорогой, но Self слишком слабый;
- дать лёгкий формат дисциплинированным людям;
- проверить спрос на недельное ревью без ежедневного сопровождения.

Предварительная рамка Lite:

| Параметр    | Черновое ограничение        |
| ----------- | --------------------------- |
| Цена        | 2 990–3 990 ₽/мес           |
| Контакт     | 1 async-ревью в неделю      |
| Ввод данных | клиент ведёт сам            |
| Анти-срыв   | нет ежедневного мониторинга |
| Продление   | куратор напоминает на ревью |

Ключевой риск: Lite может съесть время куратора почти как Pro, но принести в 2–3
раза меньше выручки. Поэтому в Фазе 0 Lite не является решением; это открытый
вопрос для отдельного прохода после первых данных по Self и Pro.

## 7. AI и Solo AI как отдельный эксперимент

AI-распознавание технически и юридически возможно, но **не нужно в Pro**.
Подробная рамка — `20`.

Solo AI логичен стратегически, но больше не нужен как основа бюджетного тарифа:
эту роль выполняет Self. На Фазе 0 AI опасен как маркетинговое обещание, потому
что:

- AI-распознавание ещё не внедрено;
- HEYS тогда начинает конкурировать с AI-трекерами на их поле;
- бренд внимательной внешней опоры размывается;
- появится новый продуктовый контур: лимиты, рекуррент, dunning, support.

В Pro AI ещё и вреден для методологии: ручной ввод еды и тренировок заставляет
куратора внимательно пройти через факты клиента, почувствовать паттерны и дать
более осмысленные рекомендации. Эти 10 минут ввода — часть продукта, а не
операционный мусор.

Правильный статус Solo AI сейчас: **R&D-гипотеза после ручной Фазы 0**, когда
будут накоплены кейсы, сценарии, скрипты и алгоритмы работы кураторов.

## 8. Метрики Фазы 0

Минимальный недельный отчёт:

| Метрика                  | Зачем                                        |
| ------------------------ | -------------------------------------------- |
| Лиды по источнику        | понять, откуда приходит подходящая аудитория |
| Лид → квалифицирован     | качество трафика, не объём                   |
| Квалифицирован → слот    | насколько оффер вызывает доверие             |
| Слот → активный триал    | готовность реально участвовать               |
| Активный триал → оплата  | главный тест оффера                          |
| Оплата → 30 дней         | тест удержания и качества                    |
| Продление по куратору    | проверка ручной помесячной оплаты            |
| Self → Pro               | показывает, работает ли Self как воронка     |
| Среднее время на клиента | база для цены Фазы 1                         |
| Причины отказа           | материал для лендинга, FAQ и Lite            |
| Инсайты клиента          | материал для кейсов и плейбука               |

Активный триал лучше считать не просто “получил доступ”, а “4+ дня участия и
достаточно данных для разбора”.

## 9. Решения и открытые вопросы

1. ✅ **Граница услуги Pro:** куратор сам вносит питание и тренировки в HEYS; от
   клиента нужны утренний чек-ин и сообщения с фактическими данными.
2. ✅ **Форма триала:** бесплатные 7 дней только после квалификации.
3. ✅ **Оплата Pro:** ручная помесячная оплата; продление предлагает куратор,
   когда видит позитивный момент и понимает контекст клиента.
4. ✅ **Внешняя подача первого набора:** не говорим про основателя. Говорим:
   первый набор со скидкой, дальше цена выше.
5. ✅ **Self:** запускаем сразу как урезанный самостоятельный режим с виджетами
   и задачником, но без куратора и инсайтов.
6. **Lite:** открытый вопрос — пока непонятно, что это за продукт и как он
   должен работать в экономике.
7. ✅ **AI:** можно делать законно и технически, но не в Pro; вернуться как к
   отдельному Self/Solo R&D после ручной Фазы 0.

Пока Lite не определён, внешний маркетинг держим в простой сетке: Self / Pro /
Pro+. Self — бюджетный самостоятельный режим; Pro — главный продукт; Pro+ —
премиум. Без обещаний AI и без полной тарифной витрины Free/Solo/Lite.
