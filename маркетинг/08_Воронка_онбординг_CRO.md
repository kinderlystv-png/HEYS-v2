## 1. Воронка целиком — этапы, рычаги, цели

| Этап             | Что происходит                   | Главный рычаг конверсии                                                          | Цель                  |
| ---------------- | -------------------------------- | -------------------------------------------------------------------------------- | --------------------- |
| Охват            | Контент/Ads ловят боль срыва     | Точность хука в боль                                                             | CTR/охват             |
| Лид              | Контакт через лид-магнит/форму   | Релевантность магнита («карта срывов»)                                           | CPL под целевым       |
| Заявка на триал  | Запись на бесплатную неделю      | Снятие трения: без карты, по записи                                              | Лид→триал ≥40%        |
| Активация триала | ≥4 активных дня, ≥12 приёмов/нед | Первый день и первый «человеческий» контакт                                      | % активных триалов    |
| Оплата           | Выбор Self/Pro/Pro+              | Опыт внимательной внешней опоры + результат недели + честная рекомендация тарифа | **Триал→оплата ≥30%** |
| Удержание        | Касания, созвон, анти-срыв       | SLA + бережность при срыве                                                       | Отток ≤8–10%/мес      |
| Реферал          | Доволен → приводит               | Реф-механика + момент «вау»                                                      | Доля рефералов        |

**Где ломается чаще всего:** активация триала (день 1) и переход триал→оплата.

## 2. Онбординг триала — подневный сценарий (тариф Pro, 7 дней)

Цель недели: клиент должен **почувствовать внимательную внешнюю опору** и
получить один осязаемый вывод о себе. Не «продать», а показать ценность.

| День          | Куратор делает                                                                                                                     | Клиент чувствует                      |
| ------------- | ---------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------- |
| **0 (старт)** | Приветствие в мессенджере, стартовая настройка целей/режима, объясняет «как слать фото за 5 сек», назначает первый утренний чек-ин | «Меня встретили, всё просто»          |
| **1**         | Первый утренний чек-ин: план/фокус дня; вечером — первый разбор присланной еды, без оценок и нравоучений                           | «За мной правда смотрят»              |
| **2–3**       | Вносит дневник, отвечает ≤30 мин в рабочее время, отмечает 👍/✅; ловит первый паттерн                                             | «Не надо считать самому»              |
| **4**         | Мини-вывод: «вот что я уже вижу по вашим данным» (1 инсайт — напр. вечерний триггер)                                               | «Узнал о себе новое»                  |
| **5–6**       | Поддержка в реальной ситуации (кафе/поездка/срыв) — момент истины анти-срыва                                                       | «В сложный момент не один»            |
| **7**         | Недельный чек-ин (созвон/аудио): саммари → что мешало → честная рекомендация тарифа (Self/Pro/Pro+) под клиента                    | «Понятно, что дальше и зачем платить» |

**Правила онбординга:**

- День 1 решает активацию — назначать старт на день, когда клиент реально
  включится.
- Никакого «отчёта задним числом» и осуждения за пропуск (это зашито в FAQ и
  тон).
- Рекомендация тарифа — **честная под клиента**, не «продай Pro+ всем»:
  дисциплинированному подойдёт Self, срывающемуся — Pro. Доверие важнее среднего
  чека на первой оплате.

### Определение активного триала (анти-абьюз + сигнал качества)

`≥4 активных дня` И `≥12 приёмов за неделю` (из брифа). Неактивный триал →
заморозка до первой активности. Активация — опережающий индикатор оплаты: её и
оптимизируем.

## 4. Win-back и удержание

| Ситуация                  | Действие                                                                         |
| ------------------------- | -------------------------------------------------------------------------------- |
| Пропал на 2–3 дня         | Куратор пишет первым, бережно, без вины; предлагает «самый простой шаг»          |
| Падение активности неделю | Мини-созвон по триггеру (Pro+), пересбор фокусов                                 |
| Не продлил после месяца   | Через 3–5 дней — тёплый чек-ин «как вы сейчас?», предложение вернуться/заморозки |
| Заморозка (форс-мажор)    | До 2 недель — удержать связь, не терять контакт                                  |
| Доволен, есть результат   | Точка для реферала: «знаете, кому это нужно?» + реф-бонус                        |

## 5. CRO-аудит реального лендинга

### Что уже сильно (не трогать)

- **Hero бьёт в боль и отстраивает:** «Верните контроль… Без диет и насилия над
  собой» + «Не один на один с задачей». Это соответствует эмоциональному ядру:
  внимательная внешняя опора вместо одиночного контроля.
- **Снятие трения у CTA:** «Без карты · Без автосписаний · Не медицинская
  услуга» — одновременно конверсия и юр-дисклеймер. Отлично.
- **Честная редкость:** блок про ограниченное число мест — правдивое объяснение
  (ёмкость кураторов), а не фейк-таймер. Премиум-аудитория это ценит.
- **FAQ глубокий и закрывает реальные страхи** (срыв, подсчёты, оплата,
  приватность).
- Страница структурно богатая: Demo → Curator → Pain → HowItWorks → Comparison →
  Navigator → Trust → Objections → форматы/тарифы/триал/FAQ.

### Что добавить / протестировать (приоритет ↓)

1. **Социальное доказательство выше по странице.** Отзывы (`SHARED_QUOTES`) есть
   в конфиге — поднять блок ближе к первому экрану (сейчас в порядке секций его
   нет в начале). Доверие к health-сервису решается рано.
2. **Явный ценовой якорь против нутрициолога.** Добавить строку «Личное ведение
   у нутрициолога — 25 000+ ₽/мес. В HEYS — кратно меньше + приложение». Сейчас
   ценность есть, а сравнения по цене с 1:1 нет.
3. **Risk-reversal крупно у тарифов.** «7 дней бесплатно, без карты» вынести как
   повторный якорь прямо в блок тарифов (не только в триал-секции).
4. **Микро-демо «как это выглядит за 5 секунд».** GIF/скрин «фото еды на весах →
   куратор внёс» — снимает страх рутины сильнее, чем текст.
5. **A/B второго экрана:** Comparison (vs трекер) против Pain — что лучше держит
   скролл. `ABTestSwitcher`/`landing-variants` уже поддерживают A/B —
   инфраструктура есть.

## 6. Контрольные конверсии (вшить в аналитику с дня 1)

`лид → заявка-на-триал → активный-триал (день 4) → оплата → продление (мес 2)`.
Считать **каждый переход** в Yandex Metrica (сервер-сайд, без трансграничной
передачи) + сверять с Trial Machine. Опережающий индикатор выручки — **%
активных триалов**, а не число лидов.
